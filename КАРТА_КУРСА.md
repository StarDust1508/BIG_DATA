## 🛤 Альтернативные траектории прохождения

Не все хотят учиться по порядку. Вот **5 траекторий** для разных целей.

### 🛤 Траектория 1: «Хочу всё с нуля» (12 недель)

```mermaid
flowchart LR
    A[Модуль 00] --> B[Модуль 01]
    B --> C[Модуль 02]
    C --> D[Модуль 03]
    D --> E[Модуль 04]
    E --> F[Модуль 05]
    F --> G[Модуль 06]
    G --> H[Модуль 07]
    H --> I[Углублённое]
```

Стандартный путь. 1–2 часа в день × 12 недель.

### 🛤 Траектория 2: «Я уже знаю Python и Pandas» (6 недель)

```mermaid
flowchart LR
    A[Модуль 00 быстро] --> B[Модуль 01]
    B --> D[Модуль 03 быстро]
    D --> E[Модуль 04]
    E --> F[Модуль 05]
    F --> G[Модуль 06]
    G --> H[Проект 1]
```

Пропускаем модуль 02. Hadoop — поверхностно.

### 🛤 Траектория 3: «Я ML-практик, хочу освоить Spark» (4 недели)

```mermaid
flowchart LR
    A[Модуль 00] --> E[Модуль 04]
    E --> F[Модуль 05]
    F --> G[Модуль 06]
    G --> SI[spark_internals]
    SI --> MLOps[mlops_углублённо]
```

Сразу к Spark. После — Internals + MLOps.

### 🛤 Траектория 4: «Готовлюсь к интервью» (3 недели)

```mermaid
flowchart LR
    A[ФИНАЛЬНЫЙ_ЭКЗАМЕН] --> B[Слабые места]
    B --> C[ресурсы/интервью]
    C --> D[глубокий_sql]
    D --> E[spark_internals]
    E --> F[Проект 1 в портфолио]
```

Self-assessment → точечно подтянуть → отрепетировать.

### 🛤 Траектория 5: «Я юрист, хочу понимать ML-команду» (2 недели)

```mermaid
flowchart LR
    A[Модуль 01] --> B[правовые_аспекты]
    B --> C[Модуль 02 EDA + обезличивание]
    C --> D[Модуль 06 объяснимость]
    D --> E[Проект 2 юр.документы]
    E --> F[mlops + Model Card]
```

Фокус на концепции и право, не на код.

### 🛤 Траектория 6: «Просто посмотреть» (1 час)

```mermaid
flowchart LR
    A[README] --> B[концептуальная_карта]
    B --> C[глоссарий]
    C --> D[реальные_кейсы]
```

Обзор за час чтения.
