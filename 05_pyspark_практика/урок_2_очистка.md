# Урок 5.2 — Очистка и нормализация в Spark

> Очистка в Pandas мы уже делали (модуль 02.3). В Spark — те же идеи, чуть другой синтаксис. И ОЧЕНЬ важное правило: «не делайте Python-циклов, всё через F.*».

---

## Часть 1. Пропуски

```python
# Сколько
df.select([F.count(F.when(F.col(c).isNull(), c)).alias(c) for c in df.columns]).show()

# Удалить
df.dropna()                                # ANY NaN
df.dropna(how="all")                        # все NaN в строке
df.dropna(subset=["email"])
df.dropna(thresh=3)                         # минимум 3 не-NaN

# Заполнить
df.fillna(0)
df.fillna({"age": 0, "city": "unknown"})
df.fillna({"amount": 0.0}, subset=["amount"])
```

⚠️ В Spark пропуски = `null` (не `NaN`). Хотя для float отличается: `NaN ≠ null`, и `F.isnan()` ≠ `F.isnull()`. На практике:

```python
from pyspark.sql.functions import isnan, isnull, when, count, col

df.select([
    count(when(isnull(c) | isnan(c) if t in ("double","float") else isnull(c), c)).alias(c)
    for c, t in df.dtypes
]).show()
```

## Часть 2. Дубликаты

```python
df.dropDuplicates()                          # полные дубли
df.dropDuplicates(["client_id"])             # по ключу — оставляет произвольную строку

# Оставить «последнюю» — сначала отсортировать, потом dropDuplicates
from pyspark.sql.window import Window

w = Window.partitionBy("client_id").orderBy(F.col("ts").desc())
df = df.withColumn("rn", F.row_number().over(w)).filter("rn = 1").drop("rn")
```

Это типовой паттерн: «оставить самую свежую запись по ключу».

## Часть 3. Нормализация строк

```python
df = (df
    .withColumn("email",     F.lower(F.trim(F.col("email"))))
    .withColumn("name",      F.initcap(F.trim(F.col("name"))))      # "иван иванов"
    .withColumn("phone_raw", F.regexp_replace(F.col("phone"), r"\D", ""))
    .withColumn("category",  F.lower(F.trim(F.col("category"))))
)

# Регулярки
df = df.withColumn("zip", F.regexp_extract(F.col("address"), r"\d{6}", 0))
```

### Маппинг значений

```python
# через CASE
df = df.withColumn("category_en",
    F.when(F.col("category") == "перевод",    "transfer")
     .when(F.col("category") == "пополнение", "deposit")
     .when(F.col("category") == "снятие",     "withdrawal")
     .otherwise("other"))

# через map (если карта большая)
from itertools import chain
mapping = {"перевод": "transfer", "пополнение": "deposit", "снятие": "withdrawal"}
mapping_expr = F.create_map([F.lit(x) for x in chain(*mapping.items())])
df = df.withColumn("category_en", mapping_expr.getItem(F.col("category")))
```

## Часть 4. Парсинг дат и таймстемпов

```python
df = df.withColumn("ts", F.to_timestamp("ts_str", "yyyy-MM-dd HH:mm:ss"))
df = df.withColumn("date_only", F.to_date("ts"))

# Извлечение
df = (df
    .withColumn("year",    F.year("ts"))
    .withColumn("month",   F.month("ts"))
    .withColumn("dow",     F.dayofweek("ts"))      # 1=воскресенье в Spark
    .withColumn("hour",    F.hour("ts"))
    .withColumn("quarter", F.quarter("ts"))
)

# Арифметика
df = df.withColumn("ts_plus_1h", F.col("ts") + F.expr("INTERVAL 1 HOUR"))
df = df.withColumn("days_ago", F.datediff(F.current_date(), F.to_date("ts")))
```

⚠️ В Spark формат дат — Java SimpleDateFormat (НЕ Python strftime). Ловкая ошибка: `"yyyy-mm-dd"` (нижний m) — это minutes, не months. Нужно `"yyyy-MM-dd"`.

## Часть 5. Числа: конвертация и валидация

```python
# Запятая → точка
df = df.withColumn("amount",
    F.regexp_replace(F.col("amount_str"), ",", ".").cast("double"))

# Очистка от пробелов / валюты
df = df.withColumn("clean_amount",
    F.regexp_replace(F.col("amount_str"), r"[^\d.]", "").cast("double"))

# Безопасный cast — иначе невалидное превратится в null
df = df.withColumn("age_int", F.col("age").cast("int"))
```

## Часть 6. Удаление выбросов

```python
# Перцентили (быстрая агрегация)
qs = df.approxQuantile("amount", [0.01, 0.99], 0.001)
lo, hi = qs[0], qs[1]
df = df.filter((F.col("amount") >= lo) & (F.col("amount") <= hi))

# Z-score через окно
from pyspark.sql.window import Window
w = Window.partitionBy(F.lit(1))     # весь датасет = одна группа
df = df.withColumn("z",
    (F.col("amount") - F.avg("amount").over(w)) / F.stddev("amount").over(w))
df = df.filter(F.abs(F.col("z")) < 3)
```

⚠️ `Window.partitionBy(F.lit(1))` — приём «глобальное окно». На больших данных это **shuffle в одну партицию** — ОПАСНО. Используйте `approxQuantile` или предварительный расчёт.

## Часть 7. DQ-проверки

Перед записью pipeline проверяет качество:

```python
def assert_data_quality(df):
    n = df.count()
    if n == 0:
        raise ValueError("Empty dataframe!")

    bad_ids = df.filter(F.col("client_id").isNull()).count()
    if bad_ids > 0:
        raise ValueError(f"Found {bad_ids} rows with null client_id")

    neg = df.filter(F.col("amount") < 0).count()
    if neg > n * 0.01:
        raise ValueError(f"Too many negative amounts: {neg}")

    print(f"✅ DQ passed: {n:,} rows")
```

В реальных проектах для DQ используют:
- **Great Expectations**.
- **Soda**.
- **Deequ** (Amazon, прямо на Spark).

## Часть 8. Schema validation

```python
from pyspark.sql.types import StructType, StructField, IntegerType, StringType, DoubleType

expected_schema = StructType([
    StructField("client_id", IntegerType(), False),
    StructField("ts", StringType(), False),
    StructField("category", StringType(), False),
    StructField("amount", DoubleType(), False),
])

assert df.schema == expected_schema, f"Schema mismatch! got: {df.schema}"
```

Если входная схема ушла «в сторону» — pipeline падает осознанно, а не молча выдаёт мусор.
