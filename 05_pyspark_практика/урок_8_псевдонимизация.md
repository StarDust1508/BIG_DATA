## Часть 1. Где в pipeline происходит обезличивание

```
EXTRACT (raw с ПДн)
    │
    ├──► [secure zone]
    │     ETL «сырых ПДн» — узкий круг лиц, audit log
    │
    ├──► ПСЕВДОНИМИЗАЦИЯ  ← сразу после extract
    │
    └──► [analytics zone]
          Все аналитики работают с псевдонимизированными
```

Принцип: **«как можно раньше отделить ПДн от аналитики»**. В Spark это первая трансформация после чтения.

## Часть 2. Хеширование с солью — встроенные функции

Spark из коробки умеет хешировать:

```python
import pyspark.sql.functions as F

SALT = "ProjectXYZ-2026-very-secret"   # ВНИМАНИЕ: см. §5 про хранение

df = df.withColumn("email_token",
    F.sha2(F.concat(F.lit(SALT), F.col("email")), 256))

# первые 16 символов для короткого токена
df = df.withColumn("email_token", F.substring(F.col("email_token"), 1, 16))

df = df.drop("email")
```

`F.sha2(col, bits)` — встроенная функция Spark. Быстрая (не Python UDF).

Альтернатива: `F.md5()` — короче (32 hex), но **криптографически слабее**. Для ПДн используйте sha2/sha256/sha512.

## Часть 3. Маппинг-таблица (обратимая псевдонимизация)

Если нужна возможность восстановить связь (по запросу или для CRM):

```python
# 1. Создать маппинг один раз
mapping = (df
    .select("client_id")
    .distinct()
    .withColumn("token",
        F.substring(F.sha2(F.concat(F.lit(SALT), F.col("client_id")), 256), 1, 16))
)

# 2. Сохранить в защищённое место
mapping.write.mode("overwrite").parquet("/secure/mapping/")

# 3. Подменить в основном датасете
df_anon = (df
    .join(mapping, "client_id")
    .drop("client_id")
    .withColumnRenamed("token", "client_token")
)
```

## Часть 4. Маскирование на стороне Spark

«Не полностью убрать, а скрыть»:

```python
# Email: оставить первую букву + домен
df = df.withColumn("email_masked",
    F.concat(
        F.substring("email", 1, 1),
        F.lit("***@"),
        F.regexp_extract("email", r"@(.+)$", 1),
    ))

# Телефон: первые 2 + последние 2
df = df.withColumn("phone_masked",
    F.concat(
        F.substring("phone", 1, 2),
        F.lit("******"),
        F.substring("phone", -2, 2),
    ))

# Карта: первые 6 + последние 4 (PCI DSS-compliant pattern)
df = df.withColumn("card_masked",
    F.concat(
        F.substring("card_pan", 1, 6),
        F.lit("******"),
        F.substring("card_pan", -4, 4),
    ))
```

## Часть 5. Хранение соли — критический момент

❌ **НЕ так:**
```python
SALT = "my-secret-salt"   # в коде → видно в Git, видно в логах
```

✅ **Правильно:**
```python
import os
SALT = os.environ["PSEUDO_SALT"]   # из env
if not SALT:
    raise ValueError("PSEUDO_SALT must be set")
```

В проде:
- HashiCorp Vault.
- AWS Secrets Manager / KMS.
- Azure Key Vault / GCP Secret Manager.
- Hadoop Credential API.

Соль:
- НЕ в коде.
- НЕ в логах.
- НЕ в файлах `*.env` в репозитории.
- Доступ ограничен жёстко (DPO + Security).
- Ротируется при инциденте.

## Часть 6. K-анонимность в Spark

```python
# 1. Огрубить квази-идентификаторы
df = (df
    .withColumn("age_band", (F.col("age") / 5).cast("int") * 5)
    .withColumn("zip_region", F.substring("zip", 1, 3))
)

# 2. Посчитать k для каждой группы
groups = df.groupBy("age_band", "zip_region", "gender").agg(
    F.count("*").alias("k")
)

# 3. Найти проблемные (k < 3)
bad_groups = groups.filter(F.col("k") < 3)
print(f"Групп с k<3: {bad_groups.count()}")

# 4. Подавить уникальные записи
keys = ["age_band", "zip_region", "gender"]
df = (df.join(groups, keys)
    .filter(F.col("k") >= 3)
    .drop("k"))
```

## Часть 7. Шифрование at rest

Spark сам не шифрует. Это делается на уровне:
- **HDFS encryption zones** (если используете HDFS).
- **S3 server-side encryption** (SSE-S3, SSE-KMS).
- **Delta Lake с собственным шифрованием**.

Проверка: при чтении `/secure/...` данные должны быть зашифрованы. Без правильного ключа — не открыть.

## Часть 9. Право на забвение — реальный сценарий

«Удалите все мои данные» → в pipeline:

1. Берём `client_id` пользователя.
2. По mapping находим `token`.
3. Удаляем из всех downstream-датасетов записи с этим `token`.
4. Удаляем строку из mapping.
5. Логируем удаление.

В голом Parquet это **перезапись партиций**. В Delta Lake — встроенный `DELETE FROM`. Поэтому Lakehouse-формат сильно проще для compliance (см. урок 5.9).

## Часть 10. Чеклист «псевдонимизация в Spark»

- [ ] Используем `F.sha2` (не md5, не Python UDF).
- [ ] Соль из env / Secret Manager.
- [ ] Маппинг (если есть) хранится отдельно.
- [ ] Прямые ПДн удалены (не маскируются, а **удаляются**).
- [ ] K-анонимность проверена (≥ 3).
- [ ] Output в encryption zone.
- [ ] Audit log пишет всё.
- [ ] Pipeline идемпотентен (одинаковый токен у одного и того же значения).
