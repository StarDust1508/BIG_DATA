# 🌍 Реальные кейсы Big Data в индустрии

> Как настоящие компании используют Spark, Kafka, Lakehouse. Что у них получилось, что — нет. Чему можно научиться.

---

## 1. Netflix — пионер data engineering

### Что они построили
- **Iceberg** — собственный Lakehouse-формат, теперь Apache.
- **Genie** — оркестратор Spark / Hadoop / Presto заданий.
- **Atlas** — система мониторинга метрик.
- **Maestro** — workflow orchestrator (заменитель Airflow для них).

### Объём
- **Десятки петабайт** в S3.
- **Миллионы** Spark-задач в день.
- Полная аналитика просмотров, рекомендаций, кодирования видео.

### Чему учиться
- **Open source как стратегия**: они выпустили Iceberg и получили вклад целой индустрии.
- **Не бояться писать свой инструмент**, если рынок не предлагает.
- Документация и [Netflix Tech Blog](https://netflixtechblog.com/) — лучший источник реальных кейсов.

## 2. Uber — стрим + ML в реальном времени

### Что они построили
- **Apache Hudi** — Lakehouse-формат с upsert (их разработка → Apache).
- **Michelangelo** — internal ML platform.
- **AresDB** — real-time analytics database (для дашбордов).
- **Spark Streaming на сотнях ТБ в час** — обработка событий поездок.

### Сценарий: ETA-прогноз
1. Водитель и пассажир запрашивают цену.
2. ML-модель прогнозирует ETA на основе:
   - Текущего трафика (real-time).
   - Истории заказов в этом районе.
   - Погоды.
   - Особенностей маршрута.
3. Ответ за 100 мс.

### Чему учиться
- **Online inference требует другой инфраструктуры**, чем batch ML.
- **Feature Store** — централизованное хранилище признаков для моделей.
- **Hudi vs Iceberg vs Delta**: разные кейсы, разные победители.
- [Uber Engineering Blog](https://eng.uber.com/) — отличный источник.

## 3. Spotify — recommendation и Lakehouse

### Что они построили
- **Annoy** — алгоритм approximate nearest neighbors (для похожих треков).
- **Luigi** — workflow orchestrator (предшественник Airflow).
- **Klio** — streaming framework на Apache Beam.
- **Полный переезд с Cassandra на BigQuery** — несколько раз.

### Сценарий: Discover Weekly
1. Каждое воскресенье миллионы пользователей получают персонализированный плейлист.
2. Pipeline:
   - Историю прослушиваний → агрегации.
   - Embeddings треков и пользователей.
   - ANN-поиск похожих.
   - Ранжирование.
3. На всё — десятки часов на Spark + GPU.

### Чему учиться
- **Не одна big bang модель**, а pipeline маленьких.
- **Эмбеддинги** + ANN — основа современных рекомендательных систем.

## 4. Airbnb — data culture и self-service

### Что они построили
- **Airflow** — теперь Apache, главный workflow orchestrator индустрии.
- **Superset** — BI инструмент (Apache).
- **Knowledge Repo** — система документации datasets.
- **Minerva** — metrics layer (как dbt + semantic layer).

### Урок про культуру
Airbnb инвестировала миллионы в **data culture**: каждый продакт-менеджер знает SQL, может сам подключиться к dataset'у. Это уменьшает узкое место «спросить дата-инженера».

### Чему учиться
- **Self-service аналитика** — давать инструменты бизнесу, не быть бутылочным горлышком.
- **Документация datasets** — Data Card / Knowledge Repo — норма.
- [Airbnb Engineering](https://airbnb.io/projects/) — куча open source.

## 5. LinkedIn — Kafka

### Что они построили
- **Kafka** — теперь Apache, мировой стандарт streaming.
- **Samza** — Stream processing framework (Apache).
- **Pinot** — real-time OLAP (Apache).
- **Brooklin** — streaming data движок.

### Масштаб
- **7 трлн сообщений в день** через Kafka.
- **Тысячи топиков**.
- **Сотни PB** в Hadoop кластере.

### Чему учиться
- **Kafka — швейцарский нож**: messaging, event sourcing, change data capture, stream processing.
- **Не каждой компании нужен такой масштаб**. Но архитектурные принципы переносятся.
- [LinkedIn Engineering](https://engineering.linkedin.com/blog).

## 6. Российские компании

### Тинькофф

- **Стэк:** Spark, Kafka, Greenplum, ClickHouse, Airflow, dbt.
- **Особенность:** одна из первых компаний в РФ с полноценным MLOps.
- **Кейсы:** real-time антифрод, кредитный скоринг, NLP для поддержки клиентов.
- [Tinkoff Stories](https://habr.com/ru/companies/tinkoff/) — статьи на Хабре.

### Сбер

- **Стэк:** Cloudera/Hortonworks-наследие → постепенный переезд на open source / собственное.
- **СберДевайсы**: голосовой ассистент Салют → Spark + ML.

- **Сберлогистика**: route optimization → Spark + GraphX.

## 7. Анти-кейсы — чему НЕ учиться

### Anti-pattern 1: «Купим Hadoop, потом разберёмся»
Множество компаний потратили миллионы на on-prem Hadoop в 2014-2018. Большинство — мигрируют в облако сейчас.
**Урок:** не покупайте инфраструктуру под технологию. Сначала понимайте задачу.

### Anti-pattern 2: «Big Data» для 100 ГБ
Куча проектов с Spark-кластерами на 5 нод для данных, которые помещаются в один Postgres. Дорого, медленно, сложно.
**Урок:** Big Data — это последний шаг, а не первый.

### Anti-pattern 3: ML без MLOps
«Обучили модель в Jupyter → дали engineer'у переписать на Java для прода». Модель работает иначе, метрики разные, переобучить нельзя.
**Урок:** с первого дня — pipeline, версионирование, monitoring.

### Anti-pattern 4: «AI — серебряная пуля»
ML-модель прогнозирует спрос на товар, но не учитывает изменения цен конкурентами. Модель неверная по дизайну, никакой Spark не поможет.
**Урок:** ML — инструмент. Понимайте проблему **до** выбора модели.

## 8. Сравнение архитектур

| Компания | Storage | Compute | Streaming | ML |
|----------|---------|---------|-----------|-----|
| Netflix | S3 + Iceberg | Spark | Kafka | Notebook + Maestro |
| Uber | HDFS + Hudi → S3 | Spark + Flink | Kafka | Michelangelo |
| Airbnb | S3 + Iceberg | Spark + Trino | Kafka | Bighead → industry |
| Spotify | GCS + BigQuery | BigQuery + Spark | Pub/Sub | TensorFlow + ANN |
| LinkedIn | HDFS + Iceberg | Spark + Trino | Kafka (native) | Frame |
| Тинькофф | Greenplum + ClickHouse | Spark | Kafka | MLflow + Airflow |
| Яндекс | YT | Spark + YQL | YDB / Logbroker | Catboost + Yandex DataSphere |

Видите паттерн? **Все идут к Lakehouse + Spark + Kafka + managed-сервисам.**

## 9. Что взять для своего проекта

Если строите Big Data с нуля **сегодня**:

```
Хранилище:    S3 + Iceberg / Delta (или Snowflake / BigQuery для DWH-стиля)
Compute:      Spark (Databricks / EMR) или dbt + Snowflake
Streaming:    Kafka + Spark Structured Streaming / Flink
Orchestration: Airflow / Dagster / Prefect
ML platform:  MLflow + Feature Store (Feast)
BI:           Superset / Metabase / Tableau / Power BI
Monitoring:   Datadog / Grafana / open source
```

Это **2026-стандарт**. На off-prem Hadoop новые проекты почти не строят.

## ✅ Заключение

Big Data — не магия. Это набор реальных инженерных решений, накопленных индустрией за 20 лет.

Главное правило: **смотрите, что делают лучшие, и адаптируйте под свой масштаб**. Не строите Hadoop-кластер, если у вас 10 ГБ. Не строите ML-модель, если справится SQL-запрос.

Учитесь у Netflix / Uber / Airbnb — но **не копируйте** их сложность. Их масштаб уникален. Ваш стэк должен соответствовать **вашему** масштабу.
