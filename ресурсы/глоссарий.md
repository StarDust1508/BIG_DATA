# 📖 Глоссарий Big Data

> Алфавитный справочник терминов с короткими определениями, примерами и ссылками на модули, где термин раскрывается подробно.

---

## A

**ACID** — свойства транзакций: Atomicity, Consistency, Isolation, Durability. Характерны для классических СУБД. В чистом Hadoop их нет. См. *BASE*. → 03

**Action (Spark)** — операция, которая запускает фактическое выполнение плана: `count()`, `collect()`, `show()`, `write()`. Противоположность — *transformation*. → 04

**Apache Spark** — фреймворк распределённых вычислений, наследник MapReduce. В 10–100× быстрее за счёт вычислений в памяти. → 04

**Avro** — строковый бинарный формат с эволюцией схемы. Хорош для Kafka. → 01.4

---

## B

**BASE** — Basically Available, Soft state, Eventual consistency. Альтернатива ACID в распределённых системах. → 03

**Batch processing** — обработка большими порциями (раз в час/день). Противоположность *streaming*. → 03, 05

**Big Data** — данные, которые не помещаются на одной машине или требуют распределённой обработки. См. *5V*. → 01.1

**Broadcast join** — оптимизация join'а, когда маленькая таблица копируется на все узлы. → 05

---

## C

**Cassandra** — NoSQL распределённая БД с фокусом на запись и горизонтальное масштабирование. → 03 (упоминание)

**Catalyst** — оптимизатор Spark SQL. Превращает ваш DataFrame-код в оптимальный план. → 04

**CAP-теорема** — в распределённой системе нельзя одновременно гарантировать Consistency, Availability и Partition tolerance. Выбираем 2 из 3. → 01.3

**Cluster** — несколько физических/виртуальных машин, работающих как одно целое. → 01, 03

**Coalesce / Repartition** — Spark-операции для изменения числа партиций. `coalesce` уменьшает без shuffle, `repartition` — с shuffle. → 05

**Columnar storage** — колоночное хранение (Parquet, ORC). Эффективно для аналитики (читаем только нужные колонки). → 01.4

---

## D

**DAG (Directed Acyclic Graph)** — план выполнения Spark-задачи. Граф трансформаций. → 04

**DataFrame** — табличная распределённая структура в Spark. С 2.0 — основной API. → 04, 05

**Data Lake** — хранилище сырых данных в произвольных форматах (часто S3 или HDFS). → 01.2

**Data Lakehouse** — гибрид Data Lake и Data Warehouse. Технологии: Delta Lake, Iceberg, Hudi. → 01.2

**Data Warehouse** — структурированное хранилище для аналитики (Snowflake, BigQuery, Redshift). → 01.2

**Driver (Spark)** — главный процесс, который держит план и координирует *executors*. → 04

---

## E

**ETL** — Extract, Transform, Load. Базовый паттерн перемещения данных. → 05

**ELT** — то же, но Transform после Load (популярнее в облаке: грузим сырое в DWH, трансформируем там). → 05

**Executor (Spark)** — рабочий процесс на узле кластера. Делает фактическую работу. → 04

**Eventual consistency** — данные синхронизируются «когда-нибудь». См. BASE. → 03

---

## F

**Faker** — Python-библиотека для генерации синтетических данных (имена, адреса, телефоны). → 02, 05

**Flink** — конкурент Spark с фокусом на потоковую обработку. → 01.2 (обзор)

---

## G

**GDPR** — General Data Protection Regulation (ЕС, 2018). Регулирует обработку персональных данных. → правовые_аспекты.md

**GraphX** — модуль Spark для графовых вычислений. → 01.2 (обзор)

---

## H

**Hadoop** — экосистема: HDFS (хранилище) + YARN (менеджер ресурсов) + MapReduce (compute). → 03

**HBase** — NoSQL поверх HDFS, для случайного доступа к большим таблицам. → 03 (обзор)

**HDFS** — Hadoop Distributed File System. Файлы разбиваются на блоки и реплицируются. → 03

**Hive** — SQL-движок поверх Hadoop. Дал начало Spark SQL. → 03 (обзор)

---

## I

**Idempotency (идемпотентность)** — операция можно повторять — результат не изменится. Критично для retry в распределёнке. → 05

---

## J

**JVM** — Java Virtual Machine. Spark работает поверх JVM, поэтому Java обязательна. → 00

**Join** — соединение двух таблиц по ключу. Самая «опасная» операция в Spark — вызывает shuffle. → 05

---

## K

**Kafka** — распределённый брокер сообщений. Стандарт де-факто для streaming. → 01.2 (обзор)

**Kappa-архитектура** — всё через streaming, никакого батча. → 01.2

---

## L

**Lambda-архитектура** — две ветки: batch (точно) и streaming (быстро). → 01.2

**Lazy evaluation** — отложенные вычисления. Spark копит план, выполняет на action. → 04

---

## M

**MapReduce** — парадигма: данные разбиваются на ключ-значение (map), потом агрегируются по ключу (reduce). Прародитель Spark. → 03

**MLlib** — модуль ML в Spark. → 06

**Master / Worker** — узлы кластера: один координатор, много исполнителей. → 03, 04

---

## N

**Namenode** — координатор HDFS. Знает, где какой блок лежит. SPOF до HA-режима. → 03

**NoSQL** — общий термин для не-реляционных БД: Cassandra, MongoDB, HBase, Redis. → 03

---

## O

**ORC** — Optimized Row Columnar. Колоночный формат, родом из Hive. → 01.4

---

## P

**Parquet** — колоночный формат, фактический стандарт в Spark. Сжатие + быстрое чтение по колонкам. → 01.4, 05

**Partition** — кусок данных, который обрабатывается одним executor'ом. Хороший партишенинг — основа производительности. → 05

**Partitioning (HDFS / table)** — раскладка данных по папкам по значению колонки (например, `/dt=2026-05-15/`). Ускоряет чтение по фильтру. → 05

**Pipeline (MLlib)** — последовательность шагов: preprocess → feature engineering → model. → 06

**Predicate pushdown** — оптимизация: фильтр «пушится» в источник, чтобы не читать лишние строки. → 04

**PySpark** — Python API Spark. → 04+

---

## R

**RDD** — Resilient Distributed Dataset. Низкоуровневая структура Spark. Сейчас в основном через DataFrame. → 04

**Replication factor** — сколько копий каждого блока в HDFS. По умолчанию 3. → 03

**Resilient (отказоустойчивый)** — система переживает падение отдельных узлов. → 01, 03

---

## S

**S3** — облачное объектное хранилище AWS. Аналоги: GCS, Azure Blob, MinIO. Часто заменяет HDFS. → 03 (обзор)

**Schema-on-read** — схема применяется при чтении (а не при записи как в SQL). → 01.4

**Schema-on-write** — схема валидируется при записи (классические БД). → 01.4

**Shuffle** — пересылка данных между executor'ами. Самая дорогая операция. → 04, 05

**Skew (перекос)** — когда одна партиция намного больше других. Замедляет всю задачу. → 05

**Spark SQL** — модуль для работы с DataFrame через SQL. → 04, 05

**SparkSession** — точка входа в Spark API. → 00, 04

**Streaming** — обработка потока событий в реальном времени. → 01.2 (обзор)

---

## T

**Transformation (Spark)** — операция, которая возвращает новый DataFrame, но НЕ выполняет вычисление сразу. Пример: `filter`, `select`, `groupBy`. → 04

---

## V

**Velocity** — скорость поступления данных. Одна из 5V. → 01.1

**Veracity** — достоверность данных. Одна из 5V. → 01.1

**Variety** — разнообразие данных. Одна из 5V. → 01.1

**Value** — ценность, которую можно извлечь. Одна из 5V. → 01.1

**Volume** — объём данных. Одна из 5V. → 01.1

---

## W

**Window function** — оконная функция (rolling average, rank, lag). В SQL и Spark. → 05

**Worker** — см. Executor. → 04

---

## Y

**YARN** — Yet Another Resource Negotiator. Менеджер ресурсов кластера Hadoop. Spark может работать на YARN. → 03
