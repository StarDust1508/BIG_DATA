## 1. Стратегии

Есть **четыре** пути работать с курсом. Можно комбинировать.

### A. Полностью локально (рекомендую для модулей 00–06)
- Установить Python, Java, PySpark.
- IDE: VS Code или PyCharm Community.
- Цена: 0 ₽.
- Плюсы: всё под рукой, никаких лимитов.
- Минусы: нужно ставить.

### B. Локально через Docker (для модуля 03 Hadoop)
- Один `docker-compose up` — и есть кластер Hadoop / Spark / Kafka.
- Цена: 0 ₽.
- Плюсы: чистая среда, легко удалить.
- Минусы: Docker нужен (тоже бесплатный).

### C. Облачные notebooks (быстро попробовать)
- **Databricks Community Edition** — настоящий Spark бесплатно.
- **Google Colab** — Jupyter в облаке, PySpark можно поставить.
- **Kaggle Notebooks** — то же самое + датасеты под рукой.
- Плюсы: ничего не ставить.
- Минусы: лимиты времени, нет персистентности (без подписки).

### D. Профессиональный путь: dev container в VS Code
- Один файл `devcontainer.json`, и у вас одинаковая среда на любой машине.
- Используется в реальных командах. После курса — точно пригодится.

В курсе мы делаем **A + B + опционально C**. Дальше разберём всё.

## 2. Локально

### 2.1. Python 3.10+

#### macOS
```bash
# Через Homebrew (если нет brew, поставьте: https://brew.sh)
brew install python@3.11
python3 --version
```

Альтернатива — **pyenv** для управления версиями (как в команде):
```bash
brew install pyenv
pyenv install 3.11.9
pyenv global 3.11.9
```

#### Linux (Ubuntu / Debian)
```bash
sudo apt update
sudo apt install -y python3 python3-pip python3-venv
python3 --version
```

#### Windows
- Скачайте с https://www.python.org/downloads/
- При установке **поставьте галочку** "Add Python to PATH".
- В PowerShell: `python --version`.

Альтернатива — **WSL 2** (Windows Subsystem for Linux). Сильно лучше для Big Data работы — у вас будет настоящий Linux. Открываете PowerShell admin:
```powershell
wsl --install -d Ubuntu-22.04
```
И дальше работаете как в Linux.

### 2.2. Java (JDK 11 или 17)

PySpark — это интерфейс к JVM-программе, без Java запустить нельзя.

#### macOS
```bash
brew install openjdk@17
# Прописать пути
echo 'export JAVA_HOME=$(/usr/libexec/java_home -v 17)' >> ~/.zshrc
echo 'export PATH="$JAVA_HOME/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc
java -version
```

#### Linux
```bash
sudo apt install -y openjdk-17-jdk
echo 'export JAVA_HOME=/usr/lib/jvm/java-17-openjdk-amd64' >> ~/.bashrc
source ~/.bashrc
java -version
```

#### Windows
- Eclipse Temurin (бесплатный OpenJDK): https://adoptium.net/temurin/releases/
- При установке отметить «Set JAVA_HOME variable».
- Перезагрузить терминал. Проверить: `java -version`.

⚠️ Spark **может ругаться на Java 21+**. Стабильнее — Java 11 или 17.

### 2.3. Виртуальное окружение Python

**Никогда не ставьте курсовые пакеты в системный Python.** Заведите venv:

```bash
cd "путь к проекту/БигДата"
python3 -m venv .venv

# Активация
# macOS/Linux:
source .venv/bin/activate
# Windows PowerShell:
.venv\Scripts\Activate.ps1
# Windows cmd:
.venv\Scripts\activate.bat
```

В активированном venv в начале строки появится `(.venv)`. Это означает, что `pip install` теперь ставит только в этот проект.

Альтернатива — **conda / miniforge**:
```bash
# https://github.com/conda-forge/miniforge — бесплатно, без лицензии Anaconda
conda create -n bigdata python=3.11
conda activate bigdata
```

Conda удобна, если нужны нативные зависимости (например, GeoPandas с GDAL).

### 2.5. Apache Spark — отдельная установка?

В большинстве случаев **НЕТ**. PySpark — это Python-пакет, который **внутри** содержит Spark. Достаточно:

```bash
pip install pyspark==3.5.0
```

И вы можете запускать `from pyspark.sql import SparkSession` без отдельной установки Spark.

**Когда нужна отдельная установка:**
- Хотите запустить standalone-кластер из нескольких машин.
- Используете `spark-submit` для отправки задач.
- Учитесь как DevOps.

Тогда:
```bash
# macOS
brew install apache-spark

# Linux: скачать tar.gz с https://spark.apache.org/downloads.html
curl -O https://archive.apache.org/dist/spark/spark-3.5.0/spark-3.5.0-bin-hadoop3.tgz
tar -xzf spark-3.5.0-bin-hadoop3.tgz
sudo mv spark-3.5.0-bin-hadoop3 /opt/spark
echo 'export SPARK_HOME=/opt/spark' >> ~/.bashrc
echo 'export PATH=$SPARK_HOME/bin:$PATH' >> ~/.bashrc
```

Для курса это **не обязательно**.

## 3. Docker

Docker — главный инструмент инженера данных. Идея: упаковываем приложение со всеми зависимостями в «контейнер», который запускается одинаково везде.

### 3.1. Установка

#### macOS / Windows
- **Docker Desktop**: https://www.docker.com/products/docker-desktop/
- Бесплатно для личного использования.
- После установки в терминале: `docker --version` и `docker ps`.

⚠️ Лицензия Docker Desktop: бесплатна для физлиц, образования, малого бизнеса (< 250 сотрудников). Для крупных — платно. Альтернативы:
- **Rancher Desktop** (бесплатно для всех): https://rancherdesktop.io/
- **Podman Desktop**: https://podman-desktop.io/
- **OrbStack** (быстрее Docker Desktop на macOS, бесплатно для личного): https://orbstack.dev/

#### Linux
```bash
sudo apt install -y docker.io docker-compose-v2
sudo usermod -aG docker $USER
# Перелогиниться, чтобы $USER попал в группу docker
docker run hello-world
```

### 3.2. Полноценный Hadoop в Docker за 5 минут

Для модуля 03. Создайте папку и файлы:

`docker-compose.yml`:
```yaml
version: "3.8"

services:
  namenode:
    image: bde2020/hadoop-namenode:2.0.0-hadoop3.2.1-java8
    container_name: namenode
    ports:
      - "9870:9870"      # Web UI
      - "9000:9000"      # RPC
    volumes:
      - hadoop_namenode:/hadoop/dfs/name
    environment:
      CLUSTER_NAME: bigdata-learn
    env_file: ./hadoop.env

  datanode:
    image: bde2020/hadoop-datanode:2.0.0-hadoop3.2.1-java8
    container_name: datanode
    volumes:
      - hadoop_datanode:/hadoop/dfs/data
    environment:
      SERVICE_PRECONDITION: "namenode:9870"
    env_file: ./hadoop.env

  resourcemanager:
    image: bde2020/hadoop-resourcemanager:2.0.0-hadoop3.2.1-java8
    container_name: rm
    ports:
      - "8088:8088"
    environment:
      SERVICE_PRECONDITION: "namenode:9870 datanode:9864"
    env_file: ./hadoop.env

  nodemanager:
    image: bde2020/hadoop-nodemanager:2.0.0-hadoop3.2.1-java8
    container_name: nm
    environment:
      SERVICE_PRECONDITION: "namenode:9870 datanode:9864 rm:8088"
    env_file: ./hadoop.env

volumes:
  hadoop_namenode:
  hadoop_datanode:
```

`hadoop.env`:
```
CORE_CONF_fs_defaultFS=hdfs://namenode:9000
CORE_CONF_hadoop_http_staticuser_user=root
HDFS_CONF_dfs_replication=1
YARN_CONF_yarn_resourcemanager_hostname=rm
```

Запуск:
```bash
docker compose up -d
docker compose ps     # все services должны быть UP
```

Зайти в namenode:
```bash
docker exec -it namenode bash
hdfs dfs -ls /
```

Web UI:
- HDFS: http://localhost:9870
- YARN: http://localhost:8088

Остановить:
```bash
docker compose down            # стоп
docker compose down -v          # стоп + удалить данные
```

### 3.3. Spark в Docker (опционально)

Если не хотите ставить Spark локально — есть готовый образ:

```bash
docker run -it --rm \
  -p 4040:4040 \
  -v "$(pwd)":/work -w /work \
  apache/spark-py:3.5.0 \
  /opt/spark/bin/pyspark
```

Это запустит PySpark внутри контейнера, ваш код будет в `/work`.

### 3.4. Kafka в Docker (бонус, для streaming в проекте 07.3)

`docker-compose.kafka.yml`:
```yaml
version: "3"
services:
  kafka:
    image: bitnami/kafka:latest
    ports:
      - "9092:9092"
    environment:
      KAFKA_CFG_NODE_ID: 0
      KAFKA_CFG_PROCESS_ROLES: controller,broker
      KAFKA_CFG_CONTROLLER_QUORUM_VOTERS: 0@kafka:9093
      KAFKA_CFG_LISTENERS: PLAINTEXT://:9092,CONTROLLER://:9093
      KAFKA_CFG_ADVERTISED_LISTENERS: PLAINTEXT://localhost:9092
      KAFKA_CFG_LISTENER_SECURITY_PROTOCOL_MAP: CONTROLLER:PLAINTEXT,PLAINTEXT:PLAINTEXT
      KAFKA_CFG_CONTROLLER_LISTENER_NAMES: CONTROLLER
      ALLOW_PLAINTEXT_LISTENER: "yes"
```

Запуск:
```bash
docker compose -f docker-compose.kafka.yml up -d
```

## 4. Облака

Реальный Spark «в облаке» бесплатно — да, это возможно.

### 4.1. Databricks Community Edition ⭐

**Лучший вариант для попробовать настоящий Spark.**

- Сайт: https://community.cloud.databricks.com/
- Регистрация бесплатна (требуется email).
- Что вы получаете: 15 ГБ кластер Spark, notebooks, рабочее место без установки.
- Лимиты: один кластер, выключается при простое.

Идеально для модулей 04–06: открыл, написал PySpark код, увидел результат.

### 4.2. Google Colab

- Сайт: https://colab.research.google.com/
- Бесплатно: 12 ГБ RAM, T4 GPU иногда даёт.
- PySpark ставится одной командой:
  ```python
  !pip install pyspark
  ```
- Лимиты: сессия 12 часов, диск не сохраняется (нужно сохранять в Google Drive).

Хорошо для модулей 02 и 06 (с GPU).

### 4.3. Kaggle Notebooks

- Сайт: https://www.kaggle.com/code
- Бесплатно: 30 ГБ RAM, GPU/TPU.
- Огромная база открытых датасетов прямо под рукой.

Идеально для практик с EDA и ML.

### 4.4. AWS / GCP / Azure free tier

- Бесплатные лимиты на годы.
- Можно поднять EMR / Dataproc на час — пока вы не превысите free tier.
- Сложно — нужна карта.

Для курса **не обязательно**. Только если хотите потрогать «настоящий» managed-Spark.

### 4.5. Yandex DataSphere

- Российский аналог Colab от Yandex Cloud.
- Бесплатные час на месяц для новых пользователей.
- PySpark поддерживается.

Подходит, если есть Yandex Cloud аккаунт.


## 8. Workflow

Как работает дата-инженер «по-взрослому»:

```
1. Создал ветку:                 git checkout -b feature/new-pipeline
2. Пишу код, коммичу часто:       git commit -m "Add ETL stage"
3. Запускаю тесты:                pytest
4. Проверяю стиль:                ruff check .
5. Запускаю pipeline локально:     python pipeline.py --run-date 2026-05-15
6. Открываю Pull Request на GitHub.
7. Code review.
8. Merge в main.
9. CI/CD деплоит в production.
```

Курс — отличное место, чтобы привыкнуть к этому циклу. С каждой практикой делайте коммит.

### pre-commit hooks (бонус)
Автоматическая проверка перед каждым коммитом:
```bash
pip install pre-commit
```
`.pre-commit-config.yaml`:
```yaml
repos:
  - repo: https://github.com/astral-sh/ruff-pre-commit
    rev: v0.4.0
    hooks:
      - id: ruff
      - id: ruff-format
```
```bash
pre-commit install
```

Теперь каждый коммит проверяет код.

## 9. Секреты

⚠️ Самый частый «провал» начинающих — закоммитить пароль в Git. **Никогда** так не делайте.

### Локально — `.env` файл
```
PSEUDO_SALT=Bz73a@9LkP3eq-NICESALT
DATABASE_URL=postgresql://user:pass@host/db
AWS_ACCESS_KEY=AKIA...
```

Загружать в Python:
```python
from dotenv import load_dotenv
import os
load_dotenv()
salt = os.environ["PSEUDO_SALT"]
```

И добавить `.env` в `.gitignore` (см. §7).

### Продакшен
- **HashiCorp Vault** — open source.
- **AWS Secrets Manager / KMS** — платно, есть free tier.
- **GCP Secret Manager / Azure Key Vault** — аналоги.
- **GitHub Actions Secrets** — для CI/CD.

### Если случайно закоммитили секрет
1. Сразу **отзовите/смените** его.
2. Удалите из истории Git через `git filter-repo` (или BFG Repo-Cleaner).
3. Force push (с осторожностью).

## 10. Troubleshooting

| Симптом | Что делать |
|---------|------------|
| `JAVA_HOME is not set` | См. §2.2, прописать переменную |
| `ModuleNotFoundError: pyspark` | Активировать venv, `pip install -r requirements.txt` |
| `winutils.exe not found` (Windows) | Скачать https://github.com/cdarlint/winutils, положить в `C:\hadoop\bin`, прописать `HADOOP_HOME` |
| Spark на macOS падает по памяти | `.config("spark.driver.memory","2g")` |
| Docker: container «exit 137» | Не хватает RAM Docker'у. Открыть Docker Desktop → Resources → дать 4+ ГБ |
| `Permission denied` (Linux Docker) | `sudo usermod -aG docker $USER`, перелогиниться |
| Установка pyarrow медленная | Использовать `uv pip install` вместо `pip` |
| Spark UI недоступен | По умолчанию http://localhost:4040 пока сессия активна |
| Jupyter не видит venv | `python -m ipykernel install --user --name bigdata` |
