# 02. Python для данных

> Прежде чем писать PySpark, надо чувствовать Pandas. Иначе PySpark получится как «Pandas с задержками», а не как настоящий распределённый код.

---

## 🎯 Цели модуля

После модуля вы:

- Свободно работаете с `pandas` и `numpy`: загрузка, очистка, агрегации, join'ы, окна.
- Делаете полноценный **EDA** с визуализациями.
- Понимаете, **когда** Pandas начинает «задыхаться» (память, скорость).
- Знаете обходные пути до Spark: `chunksize`, dtype-оптимизация, **Polars**, **Dask**, **DuckDB**.
- Применяете **псевдонимизацию ПДн** на этапе препроцессинга (152-ФЗ / GDPR).

## 🛠 Что нужно установить

Модуль работает на чистом Python — **без Java, без Spark**. Достаточно базы из модуля 00.

Проверьте, что эти пакеты доступны (они в `requirements.txt`):
```bash
python -c "import pandas, numpy, matplotlib, seaborn, sklearn; print('ok')"
```

Для практики 2 (запись Parquet) нужен `pyarrow` — он тоже в requirements.txt.

🌟 **Полностью облачно (без установки):** можете пройти весь модуль в **Google Colab** или **Kaggle Notebooks**. Эти платформы дают pandas/numpy/matplotlib «из коробки». См. [среда_разработки.md §4.2-4.3](../ресурсы/среда_разработки.md#4-облака).
