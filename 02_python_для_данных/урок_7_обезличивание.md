## Часть 1. Терминологические различия

| Термин | Что делает | Юридически |
|--------|-----------|------------|
| **Маскирование** | Закрывает часть значения (`иван***@mail.ru`) | Часто легко восстанавливается → как ПДн |
| **Псевдонимизация** | Заменяет ID на токен через хеш/таблицу | Всё ещё ПДн (152-ФЗ, GDPR) |
| **Анонимизация** | Необратимо удаляет связь с личностью | **Не ПДн** (если действительно необратимо) |
| **Шифрование** | Преобразует значение, восстановимо ключом | ПДн, но защищённые |
| **Агрегация** | Группирует так, что отдельная запись теряется | Часто = анонимизация |

> Главное: **псевдонимизация ≠ анонимизация**. Псевдонимизированные данные защищаются как ПДн.

## Часть 2. Псевдонимизация через хеш

```python
import hashlib
import pandas as pd

SECRET_SALT = "ProjectXYZ-2026-very-secret"   # хранить ОТДЕЛЬНО

def hash_value(v: str, salt: str = SECRET_SALT) -> str:
    return hashlib.sha256((salt + str(v)).encode()).hexdigest()[:16]

df["email_token"] = df["email"].apply(hash_value)
df = df.drop(columns=["email"])
```

⚠️ **Почему нужна соль?**
Без соли `sha256("ivan@mail.ru")` одно и то же у всех в мире — атака rainbow-table или брутфорс распространённых имён восстанавливает оригинал за секунды. Соль делает атаку невозможной (если соль настоящая секретная и не утекла).

⚠️ **Соль — это секрет.** Её НЕЛЬЗЯ хранить в репозитории. Берите её из переменной окружения или Secret Manager.

```python
import os

```

## Часть 3. Маппинг-таблица (обратимая псевдонимизация)

Иногда нужно потом восстанавливать связь (например, для уведомления пострадавших клиентов).

```python
# Создаём один раз
unique_ids = df["client_id"].unique()
mapping = pd.DataFrame({
    "client_id": unique_ids,
    "token": [hash_value(x) for x in unique_ids],
})
mapping.to_parquet("/secure/mapping.parquet")   # ХРАНИТЬ В ЗАЩИЩЁННОМ МЕСТЕ
```

```python
# Псевдонимизируем рабочую копию
df = df.merge(mapping, on="client_id")
df = df.drop(columns=["client_id"]).rename(columns={"token": "client_token"})
```

⚠️ Этот mapping — это **«ключ»** псевдонимизации. Его компрометация = деанонимизация всех данных. Хранится отдельно, с минимальным числом доступов.

## Часть 4. Маскирование (частичное скрытие)

Когда нужно показать **что-то**, но не всё:

```python
def mask_email(e):
    if pd.isna(e) or "@" not in e:
        return e
    name, domain = e.split("@", 1)
    return name[0] + "*" * (len(name) - 1) + "@" + domain

df["email_masked"] = df["email"].apply(mask_email)
# "ivan.petrov@mail.ru" → "i**********@mail.ru"
```

```python
def mask_phone(p):
    s = str(p)
    return s[:2] + "*" * (len(s) - 4) + s[-2:]
```

```python
def mask_inn(inn):
    s = str(inn)
    return "*" * (len(s) - 4) + s[-4:]
```

⚠️ Маскирование **не считается** надёжным обезличиванием. Если у вас есть «Иван П***** из Самары» и публичный список фамилий — деанонимизация часто тривиальна.

## Часть 6. K-анонимность

Идея: каждая запись должна быть **неотличима** от как минимум k−1 других по «квази-идентификаторам».

**Квази-идентификаторы** — поля, которые сами по себе не идентифицируют, но в комбинации могут (возраст + пол + индекс).

```python
# Простейшая проверка k-анонимности
quasi_id = ["zip", "age", "gender"]
counts = df.groupby(quasi_id).size()
print(f"min group size = {counts.min()}")    # это и есть k
print(f"non-k records  = {(counts < 3).sum()}")
```

**Что делать, если k < 3:**

1. **Обобщить (generalize):** возраст 27 → диапазон 25–29, индекс 123456 → 12345*.
2. **Подавить (suppress):** удалить редкие записи (их мало → их нельзя «спрятать в толпе»).

```python
# Огрубление
df["age_band"]    = (df["age"] // 5 * 5).astype(str) + "–" + (df["age"] // 5 * 5 + 4).astype(str)
df["zip_region"]  = df["zip"].astype(str).str[:3] + "***"
```

K-анонимность часто комбинируют с **l-разнообразием** (l разных чувствительных значений в группе) и **t-близостью** (распределение в группе близко к общему).

## Часть 7. Дифференциальная приватность (вкратце)

Самая «крутая» концепция: вместо изменения данных, мы **добавляем шум** в результаты так, что:

- Совокупная статистика правдоподобна.
- По выходу нельзя восстановить, был ли в датасете конкретный человек.

```python
def laplace_noise(value, sensitivity, epsilon):
    return value + np.random.laplace(0, sensitivity / epsilon)

# Возвращаем «зашумлённое» среднее
mean_amount = df["amount"].mean()
noisy_mean = laplace_noise(mean_amount, sensitivity=10_000, epsilon=1.0)
```

Используется Apple, Google, Microsoft в их статистике. Для нашего курса — это «дальше углубитесь сами, если интересно». Библиотеки: `diffprivlib` (от IBM), `opendp`.

## Часть 8. Полный pipeline псевдонимизации

Шаблон, который имеет смысл переиспользовать:

```python
import os, hashlib
import pandas as pd

SALT = os.environ["PSEUDO_SALT"]

def pseudonymize(df: pd.DataFrame) -> pd.DataFrame:
    df = df.copy()

    # 1. Прямые идентификаторы — захешировать
    for col in ["email", "phone", "passport", "inn"]:
        if col in df.columns:
            df[col + "_token"] = df[col].astype(str).apply(
                lambda v: hashlib.sha256((SALT + v).encode()).hexdigest()[:16]
            )
            df = df.drop(columns=[col])

    # 2. Имена удалить полностью
    for col in ["full_name", "first_name", "last_name", "middle_name"]:
        if col in df.columns:
            df = df.drop(columns=[col])

    # 3. Адрес огрубить до региона
    if "city" in df.columns:
        # keep city, drop street
        for col in ["street", "house", "apartment"]:
            if col in df.columns:
                df = df.drop(columns=[col])

    # 4. ДР огрубить до года
    if "birth_date" in df.columns:
        df["birth_year"] = pd.to_datetime(df["birth_date"]).dt.year
        df = df.drop(columns=["birth_date"])

    return df

clean = pseudonymize(raw_df)
```
