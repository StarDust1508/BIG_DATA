## Часть 1. Загрузка из разных форматов

### CSV — самое частое
```python
df = pd.read_csv(
    "file.csv",
    sep=",",                       # или ";", "\t"
    encoding="utf-8",              # "cp1251" для старых рос. файлов
    parse_dates=["ts"],            # сразу в datetime
    na_values=["", "n/a", "—"],   # что считать NaN
    dtype={"id": "int32"},         # явные типы
    nrows=1000,                    # читать только первые N строк
    chunksize=100_000,             # итератор по чанкам (для больших файлов)
)
```

### Excel
```python
df = pd.read_excel("report.xlsx", sheet_name="Sheet1", header=2)
# чтение нескольких листов
all_sheets = pd.read_excel("report.xlsx", sheet_name=None)  # dict {sheet: df}
```

### JSON
```python
df = pd.read_json("file.json")
# вложенный
df = pd.json_normalize(my_dict, record_path="items", meta=["meta_field"])
```

### SQL
```python
import sqlalchemy
engine = sqlalchemy.create_engine("postgresql://user:pwd@host/db")
df = pd.read_sql("SELECT * FROM my_table WHERE year=2025", engine)
```

### Parquet (для Big Data — стандарт)
```python
df = pd.read_parquet("file.parquet")
df = pd.read_parquet("dir/", columns=["a", "b"])   # только нужные колонки
```

## Часть 3. Типичные грязные данные

### 3.1 Пропуски (NaN, None, пустые строки, "n/a")

```python
# Сколько
df.isna().sum() / len(df) * 100      # % пропусков по колонкам

# Удалить
df.dropna()                           # все строки с любым NaN
df.dropna(subset=["email"])           # только если email пуст
df.dropna(thresh=3)                   # оставить строки, где ≥3 не-NaN

# Заполнить
df.fillna({"age": 0, "name": "unknown"})
df["price"].fillna(df["price"].median())   # медианой
df["region"] = df["region"].fillna(method="ffill")   # последним известным
```

⚠️ Удаление — не всегда правильно. Если 30% строк имеют пропуск в `phone`, но `phone` вам не нужен — почему вы выкидываете всю строку?

### 3.2 Дубликаты

```python
df.duplicated().sum()
df.duplicated(subset=["email"]).sum()
df.drop_duplicates(subset=["email"], keep="first")
```

⚠️ Часто «дубликаты» — это не точные копии, а **похожие записи** (опечатки в имени, разный регистр email). Это **fuzzy matching**, тут чистый pandas не хватит.

### 3.3 Разный регистр и пробелы

```python
df["email"] = df["email"].str.strip().str.lower()
df["name"]  = df["name"].str.title()                # "иван" → "Иван"
df["text"]  = df["text"].str.replace(r"\s+", " ", regex=True)
```

### 3.4 Парсинг дат

```python
df["ts"] = pd.to_datetime(df["ts"])                 # авто
df["ts"] = pd.to_datetime(df["ts"], format="%d.%m.%Y")
df["ts"] = pd.to_datetime(df["ts"], errors="coerce") # битые → NaT

# Извлечение
df["year"]  = df["ts"].dt.year
df["month"] = df["ts"].dt.month
df["dow"]   = df["ts"].dt.dayofweek      # 0=Пн
df["hour"]  = df["ts"].dt.hour
```

⚠️ В Pandas datetime занимает 8 байт против многих байт строки. Парсинг даты при загрузке — это **обязательно**.

### 3.5 Опечатки в категориальных колонках

Видели в практике 1: `ПЕРЕВОД` vs `перевод`. Нормализация:

```python
df["category"] = df["category"].str.lower().str.strip()

# Маппинг
mapping = {"перевод":  "transfer",
           "пополнение": "deposit",
           "снятие_наличных": "withdrawal",
           "оплата_услуг": "payment"}
df["category_en"] = df["category"].map(mapping)
```

### 3.6 Числа-строки

```python
df["amount"] = df["amount"].str.replace(",", ".").astype(float)
df["amount"] = pd.to_numeric(df["amount"], errors="coerce")
```

### 3.7 Выбросы (outliers)

```python
# Z-score
from scipy import stats
df["z"] = stats.zscore(df["amount"])
df_clean = df[df["z"].abs() < 3]

# IQR
q1, q3 = df["amount"].quantile([0.25, 0.75])
iqr = q3 - q1
df_clean = df[(df["amount"] > q1 - 1.5*iqr) & (df["amount"] < q3 + 1.5*iqr)]

# Перцентили (самое популярное)
hi = df["amount"].quantile(0.99)
df_clean = df[df["amount"] < hi]
```

⚠️ Выбросы — не всегда ошибка. Иногда это **самое интересное** (фрод, аномальные сделки). Думайте, прежде чем удалять.

## Часть 4. Кодировки

Боль номер один при работе с русскими файлами.

| Кодировка | Где встречается |
|-----------|-----------------|
| `utf-8` | Современный стандарт |
| `cp1251` (windows-1251) | Старые рос. системы, Excel из Windows |
| `koi8-r` | Очень старые |
| `utf-8-sig` | UTF-8 c BOM (часто Excel) |

```python
# Определить
import chardet
with open("file.csv", "rb") as f:
    raw = f.read(10_000)
    print(chardet.detect(raw))
```

```python
# Прочитать
df = pd.read_csv("file.csv", encoding="cp1251")
```

## Часть 5. Сохранение

```python
df.to_csv("out.csv", index=False, encoding="utf-8")
df.to_excel("out.xlsx", index=False)
df.to_parquet("out.parquet", compression="snappy")
df.to_json("out.json", orient="records", force_ascii=False)
```

В Big Data пайплайнах **выходной формат — Parquet**. CSV годится только для обмена с внешним миром.
