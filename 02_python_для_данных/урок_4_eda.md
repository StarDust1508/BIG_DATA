# Урок 2.4 — EDA: разведочный анализ данных

> EDA = Exploratory Data Analysis. Главный навык аналитика и одна из самых ценных компетенций ML-инженера.

---

## Часть 1. Зачем EDA

Прежде чем строить модель, считать метрики или принимать решения — **посмотрите на данные**. EDA отвечает на вопросы:

- Что вообще здесь есть?
- Какие у меня переменные, какого типа?
- Что распределено нормально, что — нет?
- Есть ли зависимости между переменными?
- Где «дырки», аномалии, выбросы?
- Какие гипотезы стоит проверить?

> Без EDA вы строите модель в темноте. С EDA — вы знаете, чего ожидать.

---

## Часть 2. Структура EDA-отчёта

1. **Размер и схема.** `df.shape`, `df.dtypes`.
2. **Пропуски и качество.** `df.isna().sum()`, дубликаты.
3. **Описательные статистики.** `df.describe()`, `df["x"].value_counts()`.
4. **Распределения.** Гистограммы, boxplot.
5. **Двумерный анализ.** Корреляции, scatter, групповые срезы.
6. **Временной анализ** (если есть даты). Тренды, сезонность.
7. **Категориальный анализ.** Соотношения, частоты.
8. **Гипотезы и вопросы для модели.**

## Часть 3. Описательные статистики

```python
df.describe()                          # числовые колонки
df.describe(include="all")             # включая object/category
df.describe(include="object")           # только строки/категории

df["category"].value_counts()
df["category"].value_counts(normalize=True)   # доли

df["amount"].quantile([0.01, 0.5, 0.99])
df["amount"].skew()                    # асимметрия (близко к 0 — симметрично)
df["amount"].kurt()                    # «хвостатость»
```

### Скос (skewness)

Признак, что распределение **не нормальное**:

- skew ≈ 0 → симметрично (типа возраста).
- skew > 1 → длинный правый хвост (типа доходов, цен).
- skew < -1 → длинный левый хвост (редко).

ML-модели часто требуют скос «выровнять» — через `np.log1p()` или Box-Cox.

## Часть 4. Визуализация: matplotlib + seaborn

```python
import matplotlib.pyplot as plt
import seaborn as sns

sns.set_theme()   # симпатичные дефолты
```

### Гистограмма (распределение одной переменной)
```python
sns.histplot(df["amount"], bins=50, kde=True)
plt.title("Распределение сумм")
plt.show()
```

### Boxplot (увидеть выбросы)
```python
sns.boxplot(x="category", y="amount", data=df)
```

### Scatter (две переменные)
```python
sns.scatterplot(x="age", y="income", hue="city", data=df, alpha=0.5)
```

### Линейный график (для временных рядов)
```python
df.set_index("ts")["amount"].plot()
```

### Heatmap (для корреляций)
```python
corr = df.select_dtypes("number").corr()
sns.heatmap(corr, annot=True, cmap="coolwarm", center=0)
```

### Pairplot (всё со всем)
```python
sns.pairplot(df[["age", "income", "score"]], hue="city")
```

🔥 Pairplot — мощно, но **медленно** на большом числе колонок. Только до 6–7 переменных.

## Часть 5. Корреляции

```python
df.corr()                            # Пирсон (для числовых, линейная)
df.corr(method="spearman")           # Спирман (рангам, для нелинейных)
df.corr(method="kendall")            # Кендалл (рангам, для маленьких выборок)
```

**Важно:**
- Корреляция ≠ причинность. Классика — «продажи мороженого vs утопления».
- Корреляция = 0 не значит «нет связи». Просто **линейной** нет (может быть параболическая).
- Большая корреляция между двумя признаками = мультиколлинеарность → проблема для линейных моделей.

## Часть 6. Двумерные срезы

Самый ценный приём — посмотреть **одно через другое**:

```python
# Среднее amount по category
df.groupby("category")["amount"].mean().sort_values()

# Распределение amount внутри каждой категории
sns.violinplot(x="category", y="amount", data=df)

# Pivot для удобства
df.pivot_table(
    index="city",
    columns="category",
    values="amount",
    aggfunc="mean",
)
```

## Часть 7. Временной анализ

Если есть `ts`:

```python
df["ts"] = pd.to_datetime(df["ts"])
df = df.set_index("ts")

# Тренд по дням
daily = df["amount"].resample("D").sum()
daily.plot()

# Скользящее среднее
daily.rolling(7).mean().plot()

# Сезонность: средняя по дням недели
df.groupby(df.index.dayofweek)["amount"].mean().plot.bar()

# По часам — например, активность пользователей
df.groupby(df.index.hour)["amount"].count().plot.bar()
```

## Часть 8. Бюджет EDA

> Сколько времени уделять EDA?

- На быстрый прикидочный анализ — **30 минут**.
- На полноценный отчёт перед ML — **полный день**.
- На production-grade analysis — **2–3 дня**.

Меньше — рискуете пропустить главное. Больше — впадаете в «аналитический паралич».

## Часть 10. Юридический угол

При EDA на персональных данных:

1. **Не делайте `df.head()` в notebook, который кому-то покажете** — там реальные ФИО.
2. EDA должен быть **на обезличенных данных**. Сначала pipeline псевдонимизации (урок 2.7), потом анализ.
3. Если в визуализации видна **уникальная** запись (один высокий столбец, один outlier) — это потенциальная утечка идентификации.
4. Графики и отчёты — **тоже** персональные данные, если из них можно восстановить личность.
