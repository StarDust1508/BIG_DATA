## Часть А. Теория Pandas/NumPy (8 вопросов)

**1.** Что такое `pd.Series`?
- a) Только список значений
- b) Одномерный массив с индексом и dtype
- c) Двумерная таблица
- d) Алиас `numpy.ndarray`

**2.** Что покажет `df.iloc[0]` в DataFrame с тремя строками?
- a) Первую колонку
- b) Первую строку как Series
- c) Ячейку (0,0)
- d) Ошибку, если индекс не числовой

- a) `object`
- b) `string`
- c) `category`

**4.** Что произойдёт при `np.array([[1,2,3]]) + np.array([10, 20, 30])`?
- a) Ошибка несовместимости форм
- b) `[[11, 22, 33]]` через broadcasting

- d) `[11, 21, 31]`

**5.** Что делает `df.groupby("x").transform("mean")`?
- a) Возвращает по одной строке на группу

- c) Заменяет колонку x на её среднее
- d) Делает pivot

**6.** В каком случае `merge` может **увеличить** число строк по сравнению с левой таблицей?
- a) Никогда
- b) При `how="left"` если ключи в правой неуникальны
- c) Только при `how="cross"`
- d) Только если есть NaN

**7.** Что выведет `np.where(arr > 0, arr, 0)`?
- a) True/False массив
- b) Массив, где отрицательные значения заменены на 0
- c) Только положительные значения
- d) Ошибку

**8.** `for row in df.iterrows()` — это:
- a) Самый быстрый способ обработки строк
- b) Антипаттерн в Pandas, заменять векторными операциями

## Часть В. Обезличивание и право (6 вопросов)

**15.** Псевдонимизированные данные — это:
- a) Уже не ПДн, можно использовать свободно
- b) Всё ещё ПДн, защита обязательна
- c) Только техническая мера, без правового значения
- d) То же, что анонимизация

**16.** Зачем нужна **соль** при хешировании ПДн?
- a) Для скорости
- b) Чтобы защититься от rainbow-table атак
- c) Чтобы хеш был длиннее
- d) Не нужна, sha256 достаточно

**17.** Что такое квази-идентификатор?
- a) Поле, которое **само** не идентифицирует, но в комбинации может
- b) Маскированный email
- c) Имя пользователя
- d) Хеш паспорта

**18.** При k-анонимности k=1 означает:
- a) Полная анонимизация
- b) В группе только 1 запись → она уникальна → деанонимизация
- c) Удалено 1 поле
- d) Все идентификаторы зашифрованы

**19.** Где правильно хранить mapping-таблицу (client_id ↔ token)?
- a) Рядом с обезличенным датасетом
- b) В отдельном защищённом хранилище с audit log и ограниченным доступом
- c) В Git, чтобы вся команда видела
- d) В переменной окружения

**20.** Можно ли восстановить ПДн из качественно анонимизированных данных?
- a) Да, всегда
- b) Нет, никогда
- c) Технически возможно через комбинацию с внешними источниками — поэтому 100% анонимность недостижима
- d) Только если у вас есть соль

## ✅ Ответы

<details>
<summary>Развернуть</summary>

**1. b.** Series — массив значений + индекс + dtype.

**2. b.** `iloc[0]` — первая строка как Series.

**3. c.** `category` — словарь категорий, экономит память в разы.

**4. b.** Broadcasting: вторая (1D) растягивается по строкам первой (2D).

**5. b.** `transform` возвращает результат той же формы, с групповым значением для каждой строки.

**6. b.** При `how="left"` и неуникальных ключах в правой таблице каждая совпавшая строка дублируется → строк станет больше.

**7. b.** `np.where(cond, a, b)` = «если cond, то a, иначе b», векторно.

**8. b.** `iterrows` — антипаттерн, медленный.

**9. b.** Обычно в 5–10× больше — типы расширяются (object вместо category и т.д.).

**10. c.** `inplace=True` память НЕ экономит, только усложняет код.

**11. b.** DuckDB — топ-выбор для аналитики на 1–100 ГБ Parquet с SQL.

**12. b.** `chunksize` возвращает итератор — читаем порциями.

**13. c.** Rust + параллелизм + lazy.

**14. c.** Parquet — стандарт.

**15. b.** Псевдонимизированные — ВСЁ ЕЩЁ ПДн.

**16. b.** Без соли — rainbow-table восстанавливает оригинал. Соль делает атаку невозможной.

**17. a.** Квази-идентификатор — поле, идентифицирующее в комбинации (возраст+пол+индекс).

**18. b.** k=1 = запись уникальна = деанонимизация.

**19. b.** Отдельное защищённое хранилище, audit log, минимум доступов.

**20. c.** 100%-я анонимность недостижима — деанонимизация через внешние данные возможна. Поэтому работаем с k-анонимностью + минимизацией данных.
