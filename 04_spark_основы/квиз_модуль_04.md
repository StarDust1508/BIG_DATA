## Часть А. Архитектура (7)

**1.** «Driver» в Spark — это:
- a) Главная нода кластера
- b) Управляющий процесс приложения, держит план и координирует executors
- c) Драйвер диска
- d) Spark UI

**2.** Партиция в Spark — это:
- a) Папка на HDFS
- b) Кусок данных, обрабатываемый одной task на одном core executor'а
- c) Логическая группа таблиц
- d) Тип кэширования

**3.** Что вызывает shuffle?
- a) `select`
- b) `filter`
- c) `groupBy`
- d) `withColumn`

**4.** Сколько Spark может иметь driver'ов?
- a) Один на приложение
- b) Один на кластер
- c) По одному на executor
- d) Сколько настроите

**5.** Что значит `master("local[*]")`?
- a) Запуск на кластере YARN
- b) Локальный режим, все ядра ноутбука
- c) Только один поток
- d) Без вычислений

**6.** Catalyst — это:
- a) Файловая система Spark
- b) Оптимизатор Spark SQL
- c) Альтернатива Java
- d) ML-фреймворк

**7.** При `df.cache()`:
- a) Spark сразу кэширует DataFrame
- b) Spark маркирует DataFrame; кэширование происходит на первом action
- c) DataFrame сохраняется на HDFS
- d) Падает ошибка

## Часть Б. DataFrame и SQL (8)

**8.** Каноничный современный API в Spark — это:
- a) RDD
- b) DataFrame
- c) Dataset (доступен только в Scala/Java)
- d) GraphX

**9.** `df.show()` — это:
- a) Transformation
- b) Action
- c) Конфигурация
- d) Часть `explain`

**10.** `df.filter("amount > 100")` и `df.filter(F.col("amount") > 100)`:
- a) Это разные операции
- b) Это эквивалентно
- c) Первое медленнее
- d) Второе не работает

**11.** `inferSchema=True` в `read.csv`:
- a) Бесплатно
- b) Читает файл дважды и поэтому может быть дорого
- c) Не работает
- d) Аналог Parquet

**12.** Какой формат подходит для производительных пайплайнов Spark?
- a) CSV
- b) JSON
- c) Parquet (или ORC)
- d) Excel

**13.** Broadcast join хорош, когда:
- a) Обе таблицы огромные
- b) Одна таблица ≤ ~10 МБ — копируется на executors, нет shuffle
- c) Никогда
- d) В streaming

**14.** Python UDF в Spark обычно:
- a) Быстрее встроенных функций

- c) Работают только в Pandas
- d) Не поддерживаются

**15.** `df.write.partitionBy("year", "month").parquet(...)`:
- a) Делит DataFrame на партиции в памяти
- b) Раскладывает на диск по папкам year=/month=, чтобы при чтении был partition pruning
- c) Игнорируется
- d) Конвертирует в CSV

## Часть В. Lazy и тюнинг (5)

## ✅ Ответы

1. **b.** Driver — управляющий процесс.
2. **b.** Партиция = единица параллелизма.

4. **a.** Один driver на приложение.
5. **b.** local[*] — все ядра локально.
6. **b.** Catalyst — оптимизатор SQL.
7. **b.** cache lazy, реально кэшируется на action.
8. **b.** DataFrame — стандарт.
9. **b.** show — action.
10. **b.** Эквивалентны (Catalyst разруливает оба).
11. **b.** inferSchema читает дважды.
12. **c.** Parquet/ORC — стандарт.
13. **b.** Broadcast join — для маленькой таблицы.
14. **b.** Python UDF медленнее.
15. **b.** Партиционирование на диск.
16. **b.** Lazy — план без выполнения.
17. **a.** Exchange = shuffle.
18. **b.** AQE — runtime-оптимизация.
19. **a.** Cache при множественном использовании.
20. **b.** Spark медленнее на малых данных.
