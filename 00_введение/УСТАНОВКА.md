# 🛠 Установка — для полного новичка

> Это инструкция для человека, который **никогда не открывал Terminal**.
> Каждую команду копируйте **точно так, как написана**, и нажимайте **Enter** после каждой.
> Если получили ошибку — посмотрите раздел [Если что-то пошло не так](#-если-что-то-пошло-не-так).

---

## ⚠️ Важно перед началом

### Как копировать команды правильно

🔴 **Внутри серого блока с командой — никаких комментариев.** Каждая такая строка ниже **выполняется по одной**.

🟢 **Объяснение** к команде всегда написано **до** или **после** серого блока, обычным текстом.

Например:

> Установить Python 3.11 через Homebrew:

```
brew install python@3.11
```

> После установки проверить версию:

```
python3 --version
```

Не пытайтесь скопировать сразу 5 команд из 5 блоков — копируйте **по одной**, нажимайте Enter, ждите завершения, потом следующую.

### Что значит `$` или `%` в выводе

Когда видите в Terminal что-то вроде:

```
egorsabalin@MacBook-Air ~ %
```

Это **приглашение к вводу**. Знак `%` (или `$`) — НЕ копируется. Это просто значит «терминал ждёт команду».

## Шаг 1 — Откройте Terminal

На Mac:
- `Cmd + Space` → введите `Terminal` → Enter.
- Откроется чёрное окошко.

Не закрывайте его — все команды будете вводить туда.

---

## Шаг 2 — Установите Homebrew

Homebrew — это «магазин программ» для Mac. Без него установка Python и Java — мука.

Проверьте сначала, может Homebrew уже стоит:

```
brew --version
```

Если показывает версию (например, `Homebrew 4.x.x`) — переходите к шагу 3.

Если пишет `command not found: brew` — установите его. Скопируйте **одну** длинную команду:

```
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

Установщик спросит пароль от Mac — введите, ничего не покажет (это нормально, пароль вводится «вслепую») — нажмите Enter.

Установщик скажет что-то типа:

```
==> Next steps:
- Run these two commands in your terminal to add Homebrew to your PATH:
  echo >> /Users/USER/.zprofile
  echo 'eval "$(/opt/homebrew/bin/brew shellenv)"' >> /Users/USER/.zprofile
  eval "$(/opt/homebrew/bin/brew shellenv)"
```

**Скопируйте именно эти три команды**, которые ВАМ написал установщик (там будет ваше имя пользователя) — это привяжет Homebrew к PATH.

Проверьте:

```
brew --version
```

Если показывает версию — отлично.

## Шаг 3 — Установите Python 3.11

```
brew install python@3.11
```

Подождите 1–3 минуты. В конце скажет:

```
🍺 /opt/homebrew/Cellar/python@3.11/...
```

⚠️ **Важно для Mac:** на новых Mac в системе уже есть старый Python 3.9. После установки Python 3.11 команда `python3` может показывать **старый** Python. Это нормально, мы это сейчас починим.

Проверьте, какая версия используется:

```
python3 --version
```

Если показывает **3.11.x** — отлично.

Если показывает **3.9.x** или другое — нужно прописать PATH. Выполните:

```
echo 'export PATH="/opt/homebrew/opt/python@3.11/libexec/bin:$PATH"' >> ~/.zshrc
```

Потом:

```
source ~/.zshrc
```

Потом снова:

```
python3 --version
```

Теперь должно быть 3.11.x.

## Шаг 4 — Установите Java 17

```
brew install openjdk@17
```

Подождите 3–5 минут.

После установки нужно «зарегистрировать» Java в системе. Скопируйте **точно эту команду** (она длинная, в одну строку):

```
sudo ln -sfn /opt/homebrew/opt/openjdk@17/libexec/openjdk.jdk /Library/Java/JavaVirtualMachines/openjdk-17.jdk
```

Спросит пароль Mac — введите (вслепую) — Enter.

Теперь прописать переменную JAVA_HOME. Скопируйте три команды **по одной**:

```
echo 'export JAVA_HOME=$(/usr/libexec/java_home -v 17)' >> ~/.zshrc
```

```
echo 'export PATH="$JAVA_HOME/bin:$PATH"' >> ~/.zshrc
```

```
source ~/.zshrc
```

Проверьте, что Java работает:

```
java -version
```

Должно быть что-то типа:

```
openjdk version "17.0.x" ...
```

Если показывает 17.x — отлично.

## Шаг 6 — Создайте виртуальное окружение Python

Виртуальное окружение (venv) — это «песочница» для пакетов курса, чтобы они не сломали другие ваши Python-проекты.

Создание:

```
python3 -m venv .venv
```

Подождите 10 секунд. Активация:

```
source .venv/bin/activate
```

После активации в начале строки терминала появится `(.venv)`:

```
(.venv) egorsabalin@MacBook-Air BIG_DATA %
```

Это значит — окружение активно. **Каждый раз** когда вы открываете Terminal заново, нужно:

```
cd ~/Desktop/BIG_DATA
source .venv/bin/activate
```

## ❓ Если что-то пошло не так

### `zsh: command not found: #`

Вы скопировали в Terminal строку, начинающуюся с `#`. Эти строки — комментарии в инструкциях, **их копировать не нужно**. Копируйте только сами команды.

### `zsh: command not found: brew`

Homebrew не установлен или не прописан в PATH. Вернитесь к **Шагу 2**.

### `command not found: java`

JAVA_HOME не прописана. Выполните ещё раз:

```
echo 'export JAVA_HOME=$(/usr/libexec/java_home -v 17)' >> ~/.zshrc
```

```
source ~/.zshrc
```

```
java -version
```

### `python3 --version` показывает 3.9 вместо 3.11

Старый системный Python в PATH стоит раньше нового. Выполните:

```
echo 'export PATH="/opt/homebrew/opt/python@3.11/libexec/bin:$PATH"' >> ~/.zshrc
```

```
source ~/.zshrc
```

```
python3 --version
```

### `cd: no such file or directory: путь к Desktop/БигДата`

Вы скопировали **плейсхолдер** — текст «путь к Desktop/БигДата» — буквально. Не делайте этого. Используйте конкретные команды из инструкции выше, например:

```
cd ~/Desktop/BIG_DATA
```

### `Could not open requirements file`

Вы выполняете `pip install -r requirements.txt` **не из папки курса**. Сначала зайдите в папку:

```
cd ~/Desktop/BIG_DATA
```

Потом проверьте, что файл есть:

```
ls requirements.txt
```

Если файл показывается — выполните `pip install -r requirements.txt`. Если нет — вы не в той папке.

### `Permission denied` при sudo

Mac не показывает символы пароля при вводе. Это нормально — печатайте «вслепую» и нажимайте Enter.

### `Operation not permitted` (Mac)

Иногда нужно разрешить Terminal доступ к Disk. System Preferences → Privacy & Security → Full Disk Access → ✅ Terminal.
