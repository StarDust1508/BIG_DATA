## 1. Python 3.10+

Spark нормально работает с Python 3.9–3.11. Берём 3.11 — стабильная и быстрая.

### macOS
```bash
# Если нет Homebrew, поставьте его одной командой с https://brew.sh
brew install python@3.11
python3 --version   # должно показать 3.11.x
```

Альтернатива (рекомендую для профи) — **pyenv** для управления версиями Python:
```bash
brew install pyenv
echo 'eval "$(pyenv init -)"' >> ~/.zshrc
source ~/.zshrc
pyenv install 3.11.9
pyenv global 3.11.9
```

### Linux (Ubuntu / Debian)
```bash
sudo apt update
sudo apt install -y python3.11 python3-pip python3-venv
python3 --version
```

### Windows
1. https://www.python.org/downloads/windows/ → последний Python 3.11.x.
2. При установке поставьте галочку **«Add Python to PATH»** ⚠️ важно!
3. Откройте PowerShell, проверьте: `python --version`.

**Совет для Windows:** для серьёзной работы с Big Data лучше установить **WSL 2 (Windows Subsystem for Linux)**. У вас будет настоящий Ubuntu внутри Windows.
```powershell
# В PowerShell от админа
wsl --install -d Ubuntu-22.04
```
После перезагрузки в Start menu найдёте «Ubuntu» — это полноценный Linux. Дальше следуете инструкциям Linux, а не Windows.

## 2. Java 17

PySpark — это интерфейс к JVM-программе. Без Java запустить нельзя.

⚠️ Spark **может ругаться на Java 21+**. Стабильнее всего — **Java 17** (LTS).

### macOS
```bash
brew install openjdk@17

# Зарегистрировать в системе
sudo ln -sfn $(brew --prefix)/opt/openjdk@17/libexec/openjdk.jdk \
    /Library/Java/JavaVirtualMachines/openjdk-17.jdk

# Прописать JAVA_HOME
echo 'export JAVA_HOME=$(/usr/libexec/java_home -v 17)' >> ~/.zshrc
echo 'export PATH=$JAVA_HOME/bin:$PATH' >> ~/.zshrc
source ~/.zshrc

# Проверка
java -version
# должно показать: openjdk version "17.0.x" ...
```

### Linux (Ubuntu / Debian)
```bash
sudo apt install -y openjdk-17-jdk

# Прописать JAVA_HOME
echo 'export JAVA_HOME=/usr/lib/jvm/java-17-openjdk-amd64' >> ~/.bashrc
echo 'export PATH=$JAVA_HOME/bin:$PATH' >> ~/.bashrc
source ~/.bashrc

java -version
```

### Windows
1. Скачайте **Eclipse Temurin 17** (бесплатный OpenJDK с MSI): https://adoptium.net/temurin/releases/
2. При установке отметьте:
   - ✅ Set JAVA_HOME variable
   - ✅ JavaSoft (Oracle) registry keys
3. Перезагрузите PowerShell. Проверка:
   ```powershell
   java -version
   echo $env:JAVA_HOME
   ```

## 3. Виртуальное окружение

Это нужно, чтобы пакеты курса не мешали другим вашим проектам. **Никогда не ставьте курс в системный Python.**

В терминале откройте папку курса:
```bash
cd "путь к Desktop/БигДата"
```

Создайте venv:
```bash
python3 -m venv .venv
```

Активируйте:
```bash
# macOS / Linux:
source .venv/bin/activate

# Windows PowerShell:
.venv\Scripts\Activate.ps1
# Если PowerShell ругается на политики:
# Set-ExecutionPolicy -Scope CurrentUser RemoteSigned

# Windows cmd:
.venv\Scripts\activate.bat
```

В начале строки появится `(.venv)` — значит, окружение активно.

⚠️ **Каждый раз** при открытии терминала повторяйте `source .venv/bin/activate`. Это норма.

## 4. Установка пакетов курса

```bash
pip install --upgrade pip
pip install -r requirements.txt
```

Это может занять 2–5 минут. Будет поставлено:
- `pyspark==3.5.x` — главный пакет курса (включает Spark).
- `pandas`, `numpy`, `matplotlib`, `seaborn` — анализ и графики.
- `scikit-learn` — ML вне Spark.
- `pyarrow`, `fastparquet` — Parquet.
- `jupyter` — ноутбуки.
- `faker` — генерация тестовых данных.
- ~10 дополнительных утилит.

Если установка идёт **очень медленно** — попробуйте `uv`, ускоритель в 10–100×:
```bash
pip install uv
uv pip install -r requirements.txt
```

## 7. IDE: чем редактировать код

### Вариант A — VS Code (рекомендую)

1. Скачайте https://code.visualstudio.com/ — бесплатно.
2. Откройте папку курса: `File → Open Folder → БигДата`.
3. Установите расширения (Ctrl/⌘ + Shift + X):
   - **Python** (Microsoft) — обязательно.
   - **Jupyter** — для ноутбуков.
   - **Ruff** — линтер.
   - **GitLens** — Git.
4. Внизу справа выберите интерпретатор: `.venv/bin/python`.

### Вариант B — PyCharm Community

1. https://www.jetbrains.com/pycharm/download/ — **Community Edition**, бесплатно.
2. New Project → Existing → выберите папку БигДата.
3. File → Settings → Project → Python Interpreter → Add → Existing → `.venv/bin/python`.

### Вариант C — Jupyter Lab

Если предпочитаете ноутбуки:
```bash
jupyter lab
```
Откроется в браузере http://localhost:8888.

Чтобы Jupyter «увидел» ваш venv:
```bash
python -m ipykernel install --user --name bigdata --display-name "Python (BigData)"
```

## ❓ Частые проблемы

| Симптом | Решение |
|---------|---------|
| `JAVA_HOME is not set` | Прописать переменную (см. §2). На macOS перезагрузить терминал. |
| `command not found: java` | Java не установлена или нет в PATH. См. §2. |
| `ModuleNotFoundError: pyspark` | Забыли `source .venv/bin/activate`. И `pip install -r requirements.txt`. |
| `ModuleNotFoundError: pandas` | То же — активируйте venv. |
| `winutils.exe not found` (Windows) | Скачать https://github.com/cdarlint/winutils, скопировать в `C:\hadoop\bin`, прописать переменную `HADOOP_HOME=C:\hadoop`. |
| `PermissionError` на macOS | `xattr -dr com.apple.quarantine ~/Library/Caches/pip` |
| Spark «Out of Memory» при старте | `.config("spark.driver.memory","2g")` (это уже есть в hello_spark.py) |
| Очень долгая установка `pyarrow` | Используйте `uv pip install -r requirements.txt` вместо `pip` |
| Jupyter не видит библиотеки | `python -m ipykernel install --user --name bigdata` |
| Spark UI «connection refused» | UI доступен ТОЛЬКО пока есть активная SparkSession. Откройте http://localhost:4040 после `getOrCreate()`. |

Если что-то не из этой таблицы — посмотрите в [ресурсы/среда_разработки.md §10](../ресурсы/среда_разработки.md#10-troubleshooting).
