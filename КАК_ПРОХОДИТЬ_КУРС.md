# 📖 Как проходить курс — пошагово

> Один лист на все случаи. Открыли модуль → не знаете, что делать → читайте этот файл.

---

## 🎯 Из чего состоит каждый модуль

В папке каждого модуля (`01_основы_BigData`, `02_python_для_данных` и т.д.) лежит:

```
01_основы_BigData/
├── README.md                  ← начинаете отсюда
├── урок_1_что_такое_bigdata.md   ← теория
├── урок_2_экосистема.md
├── урок_3_...
├── урок_4_...
├── практика_1_анализ_5V.py    ← код-задание
├── решение_1_анализ_5V.py     ← НЕ открывать пока не сделали сами
├── моё_решение_1.py           ← сюда вы пишете свой код (если файла нет — создайте)
├── квиз_модуль_01.md          ← проверка в конце
└── мои_заметки.md             ← ваш конспект
```

---

## ⏱ Один модуль = 3-7 дней по часу-полтора

Не пытайтесь пройти модуль за вечер. Распределите.

---

## 📚 Шаг 1: Прочитайте README модуля

В каждом модуле сверху есть `README.md`. Это **карта**: что внутри, зачем, в каком порядке.

Откройте на GitHub или в VS Code:

```
01_основы_BigData/README.md
```

Прочитайте за 5 минут. Узнаете, что будете уметь после.

---

## 📖 Шаг 2: Уроки — теория

Файлы вида `урок_N_*.md` — это теория. Читайте по порядку, **не пропускайте**.

### Как читать урок

1. Открыли файл `урок_1_что_такое_bigdata.md`.
2. Читаете медленно. Если непонятный термин — смотрите [глоссарий](./ресурсы/глоссарий.md).
3. Внизу урока обычно есть «✅ Самопроверка» — попробуйте ответить.
4. Не получается — перечитайте раздел.

⏱ **Один урок = 15-40 минут.** Если читаете дольше — устали, сделайте паузу.

---

## 🛠 Шаг 3: Практика — самое важное

После каждой группы уроков идёт практика. Файлы `практика_N_*.py`.

### Как делать практику (пошагово)

#### 3.1. Откройте Terminal и подготовьтесь

```
cd ~/Desktop/BIG_DATA
```

```
source .venv/bin/activate
```

В начале строки должно появиться `(.venv)`. Если нет — venv не активен, см. [УСТАНОВКА](./00_введение/УСТАНОВКА.md).

#### 3.2. Запустите файл практики **один раз**

```
python 01_основы_BigData/практика_1_анализ_5V.py
```

Что произойдёт:
- Скрипт сгенерирует тестовые данные (CSV в папке `datasets/`).
- Покажет список заданий (TODO 1, TODO 2, ...).
- Закроется.

#### 3.3. Откройте VS Code (или любой редактор)

```
code .
```

(если не установлен VS Code — откройте папку через Finder в любом редакторе).

#### 3.4. Найдите файл «моё_решение_N.py»

В папке модуля должен быть файл `моё_решение_N.py` с шаблоном-каркасом. Если его нет:

```
cp 01_основы_BigData/моё_решение_шаблон.py 01_основы_BigData/моё_решение_1.py
```

(копия шаблона создаст рабочий файл).

#### 3.5. Заполняйте TODO по одному

В файле увидите:

```python
# TODO 1: Прочитайте CSV-файл в DataFrame
# Подсказка: pd.read_csv('путь к файлу')
df = ...   # ← напишите ваш код здесь
```

Замените `...` на ваш код:

```python
df = pd.read_csv('datasets/transactions_sample.csv')
```

#### 3.6. Сохраните файл (Cmd+S) и запустите

В Terminal:

```
python 01_основы_BigData/моё_решение_1.py
```

- Если есть **ошибка** — прочитайте текст ошибки, найдите её в [TROUBLESHOOTING.md](./ресурсы/TROUBLESHOOTING.md), исправьте, запустите снова.
- Если **работает** — переходите к следующему TODO.

#### 3.7. Повторите для всех TODO

По одному. После каждого — запускайте и проверяйте.

⏱ **Одна практика = 30-90 минут.** Не торопитесь.

---

## 🔍 Шаг 4: Сравните с эталоном

После того как все TODO сделаны и скрипт работает — откройте файл `решение_N_*.py` (он рядом с практикой).

Сравните:
- Какой подход у вас — какой у эталона.
- Что эталон делает лучше.
- Что эталон делает по-другому.

Не страшно, если эталон лаконичнее — в начале так у всех.

---

## ✅ Шаг 5: Квиз

После всего модуля — `квиз_модуль_N.md`. Это файл с 20-25 вопросами.

1. Откройте файл.
2. Читайте вопросы по порядку.
3. Отвечайте **сами**, не подсматривая.
4. В конце файла под «Развернуть ответы» — правильные ответы.

⏱ **Квиз = 20-30 минут.**

Если набрали < 70% — вернитесь к слабым урокам. Не страшно.

---

## 📓 Шаг 6: Записывайте в «мои_заметки.md»

В каждом модуле есть пустой `мои_заметки.md`. Туда:
- Что узнали нового.
- Что было непонятно.
- Вопросы, на которые хотите вернуться.

Это **ваш** конспект — никаких правил, пишите свободно.

## 🆘 Если что-то не работает

| Проблема | Решение |
|----------|---------|
| Python ошибка | [TROUBLESHOOTING.md](./ресурсы/TROUBLESHOOTING.md) |
| Не знаю что делать в TODO | Откройте `решение_N_*.py` — посмотрите как там |
| Запутался в терминах | [Глоссарий](./ресурсы/глоссарий.md) |
| Не понимаю, что писать | Напишите [Issue](https://github.com/StarDust1508/BIG_DATA/issues) |
| Хочу пропустить модуль | Не рекомендую, но можно — будет сложнее дальше |

## 🎯 Главное правило

> **Не читать → делать. Не делать → ничего не выучить.**

Курс работает только если **руки на клавиатуре**. Читать без кода — теряете 80% пользы.
