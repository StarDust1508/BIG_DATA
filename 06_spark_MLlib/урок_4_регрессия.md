# Урок 6.4 — Регрессия

> Регрессия = «предсказываем число». Цены, доходы, объёмы. В MLlib API почти зеркальный с классификацией.

---

## Часть 1. Какие модели есть

| Модель | Когда |
|--------|-------|
| **LinearRegression** | Базовая. Хорошо для линейных зависимостей. |
| **GeneralizedLinearRegression** | Регрессии с разным распределением (Poisson, Gamma) |
| **DecisionTreeRegressor** | Нелинейные зависимости, простой single tree |
| **RandomForestRegressor** | Ансамбль RF |
| **GBTRegressor** | Gradient Boosting |

---

## Часть 2. Линейная регрессия

```python
from pyspark.ml.regression import LinearRegression

lr = LinearRegression(
    featuresCol="features",
    labelCol="amount",
    maxIter=100,
    regParam=0.01,
    elasticNetParam=0.0,
)
model = lr.fit(train)
pred = model.transform(test)
pred.select("amount", "prediction").show(5)

print("Coef:", model.coefficients)
print("Intercept:", model.intercept)
print("R^2 train:", model.summary.r2)
print("RMSE train:", model.summary.rootMeanSquaredError)
```

`model.summary` — содержит метрики на train. Для test — используем Evaluator.

## Часть 4. Метрики

```python
from pyspark.ml.evaluation import RegressionEvaluator

ev_rmse = RegressionEvaluator(labelCol="amount", metricName="rmse")
ev_mae  = RegressionEvaluator(labelCol="amount", metricName="mae")
ev_r2   = RegressionEvaluator(labelCol="amount", metricName="r2")

print(f"RMSE = {ev_rmse.evaluate(pred):.2f}")
print(f"MAE  = {ev_mae.evaluate(pred):.2f}")
print(f"R²   = {ev_r2.evaluate(pred):.4f}")
```

| Метрика | Что значит | Когда брать |
|---------|------------|-------------|
| RMSE | sqrt(средний квадрат ошибки) | По умолчанию, штрафует крупные ошибки |
| MAE | средняя абсолютная ошибка | Более устойчивая к выбросам |
| R² | доля объяснённой дисперсии (0..1) | Сравнение моделей |
| MAPE | средняя % ошибка | Когда масштаб важен |

## Часть 5. Преобразования таргета

Если target сильно скошен (типичная цена, доход) — модель плохо себя ведёт. Решение:

```python
# Перед обучением
df = df.withColumn("log_amount", F.log1p("amount"))

# Используем log_amount как labelCol
lr = LinearRegression(featuresCol="features", labelCol="log_amount")
model = lr.fit(train)

# При предсказании — обратное преобразование
pred = (model.transform(test)
    .withColumn("prediction_real", F.expm1("prediction")))
```

`log1p(x) = log(1+x)` — для положительных значений, безопасен на 0.

---

## Часть 6. Когда регрессия НЕ подходит

- Дискретный таргет (1, 2, 3 — это не «число») → классификация.
- Очень сильная нелинейность + мало данных → попробуйте дерево или признаковую инженерию.
- Цензурированные / временные данные → survival analysis, time series.
