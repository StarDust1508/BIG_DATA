# Урок 6.5 — Кластеризация: KMeans и не только

> Unsupervised learning: ярлыков нет, ищем структуру в данных. В MLlib — KMeans, BisectingKMeans, GaussianMixture, LDA.

---

## Часть 1. KMeans — самая используемая

### Идея
Разделить точки на k групп так, чтобы внутри группы они были «близкими», между группами — «далёкими».

### Код
```python
from pyspark.ml.clustering import KMeans

km = KMeans(featuresCol="features", k=5, seed=42, maxIter=20)
model = km.fit(df)

# Центры
print("Cluster centers:")
for i, center in enumerate(model.clusterCenters()):
    print(f"  Cluster {i}: {center}")

# Назначения
predicted = model.transform(df)
predicted.groupBy("prediction").count().orderBy("prediction").show()
```

### ⚠️ Перед KMeans — обязательно scaling
KMeans использует евклидово расстояние. Если возраст в [0..100], а доход в [10000..1000000] — доход «победит» в любой метрике.

```python
from pyspark.ml.feature import StandardScaler
from pyspark.ml import Pipeline

scaler = StandardScaler(inputCol="features_raw", outputCol="features")
km = KMeans(featuresCol="features", k=5)
pipe = Pipeline(stages=[scaler, km])
```

## Часть 2. Как выбрать k

### Метод «локтя» (Elbow)

```python
costs = []
for k in range(2, 11):
    m = KMeans(featuresCol="features", k=k).fit(df)
    costs.append((k, m.summary.trainingCost))

for k, cost in costs:
    print(f"k={k:2d}  WSSE={cost:.2f}")
```

Ищите «локоть» — точку, после которой улучшения резко замедляются.

### Silhouette score
```python
from pyspark.ml.evaluation import ClusteringEvaluator

ev = ClusteringEvaluator(featuresCol="features", metricName="silhouette")
silhouette = ev.evaluate(predicted)
print(f"Silhouette: {silhouette:.4f}")    # ближе к 1 = лучше
```

## Часть 3. BisectingKMeans

Иерархический вариант: всё дерево деления. Часто даёт лучшие кластеры.

```python
from pyspark.ml.clustering import BisectingKMeans

bk = BisectingKMeans(featuresCol="features", k=5, seed=42)
```

## Часть 4. GaussianMixture

Точки могут принадлежать **нескольким** кластерам с вероятностями.

```python
from pyspark.ml.clustering import GaussianMixture

gm = GaussianMixture(featuresCol="features", k=5)
model = gm.fit(df)
pred = model.transform(df)
pred.select("features", "prediction", "probability").show(5)
```

## Часть 6. Кейс: сегментация клиентов

Классическая бизнес-задача. Хотим разделить клиентов на «массовый», «mass-affluent», «premium» **без** заранее заданных меток.

```python
# Признаки клиентов
features_cols = ["monthly_income", "tx_count", "avg_tx_amount", "tx_categories_distinct"]

asm = VectorAssembler(inputCols=features_cols, outputCol="features_raw")
sc  = StandardScaler(inputCol="features_raw", outputCol="features")
km  = KMeans(featuresCol="features", k=3, seed=42)

model = Pipeline(stages=[asm, sc, km]).fit(clients_aggr)
segments = model.transform(clients_aggr)

# Описание сегментов
segments.groupBy("prediction").agg(
    F.count("*").alias("n"),
    F.round(F.avg("monthly_income"), 0).alias("avg_income"),
    F.round(F.avg("tx_count"), 1).alias("avg_tx"),
).orderBy("prediction").show()
```

## Часть 7. Юридический угол

Кластеризация на ПДн = профайлинг. По GDPR (Art. 22) автоматическое профилирование, имеющее **юридические последствия**, требует:
- информирования субъекта;
- права на human review;
- объяснимости.

Технически — описывайте кластеры (центроиды + интерпретации), не используйте профили для «секретного» targeting'а без согласия.
