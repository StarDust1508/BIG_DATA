# Урок 6.3 — Классификация: Logistic Regression, Random Forest, GBT

> Три «рабочих лошадки» классификации. Каждая в MLlib работает идиоматически одинаково.

---

## Часть 1. Логистическая регрессия

### Когда брать
- База, всегда первая модель.
- Линейная зависимость в feature-space → высокая интерпретируемость.
- Очень быстрая.
- Хорошо работает на разреженных данных (текст, OHE).

### Код
```python
from pyspark.ml.classification import LogisticRegression

lr = LogisticRegression(
    featuresCol="features",
    labelCol="label",
    maxIter=100,
    regParam=0.01,           # L2 регуляризация (или сочетание с L1)
    elasticNetParam=0.0,      # 0 = чистый L2, 1 = чистый L1
    family="binomial",        # binomial | multinomial
)

model = lr.fit(train)
pred = model.transform(test)
pred.select("label", "prediction", "probability").show(5)
```

### Выход
- `prediction` — предсказанный класс.
- `probability` — вектор вероятностей по классам.
- `rawPrediction` — логиты.

### Интерпретация
```python
print("Coef:", model.coefficients)
print("Intercept:", model.intercept)
```

В отличие от деревьев, у логрег **каждый признак имеет коэффициент** → можно сказать «увеличение возраста на 1 повышает вероятность ...».

## Часть 2. Random Forest

### Когда брать
- Без feature engineering из коробки даёт сильный результат.
- Робастен к шуму и масштабам (не нужен scaler!).
- Хорошо для разнообразных таблиц.

### Код
```python
from pyspark.ml.classification import RandomForestClassifier

rf = RandomForestClassifier(
    featuresCol="features",
    labelCol="label",
    numTrees=100,
    maxDepth=8,
    minInstancesPerNode=10,
    seed=42,
)

model = rf.fit(train)
```

### Feature importance
```python
imp = list(zip(feature_names, model.featureImportances.toArray()))
imp.sort(key=lambda x: -x[1])
for name, val in imp[:10]:
    print(f"  {name:30s} {val:.4f}")
```

## Часть 3. Gradient Boosted Trees

### Когда брать
- Часто **самая точная** модель из MLlib.
- Чуть медленнее RF при обучении.
- В Spark — только бинарная классификация (multiclass нет).

### Код
```python
from pyspark.ml.classification import GBTClassifier

gbt = GBTClassifier(
    featuresCol="features",
    labelCol="label",
    maxIter=100,
    maxDepth=5,
    stepSize=0.1,
)
model = gbt.fit(train)
```

Для **multiclass** в Spark — берите RF, или XGBoost/LightGBM через интеграцию (вне коробки).

## Часть 4. Оценка качества

### Бинарная классификация
```python
from pyspark.ml.evaluation import BinaryClassificationEvaluator

ev = BinaryClassificationEvaluator(
    labelCol="label",
    metricName="areaUnderROC",   # или "areaUnderPR"
)
auc = ev.evaluate(pred)
print(f"ROC-AUC: {auc:.4f}")
```

### Многоклассовая
```python
from pyspark.ml.evaluation import MulticlassClassificationEvaluator

ev = MulticlassClassificationEvaluator(
    labelCol="label",
    metricName="f1",     # f1 | accuracy | weightedPrecision | weightedRecall
)
f1 = ev.evaluate(pred)
print(f"F1: {f1:.4f}")
```

### Confusion matrix вручную
```python
pred.groupBy("label", "prediction").count().show()
```

## Часть 5. Балансировка классов

Большинство реальных задач — несбалансированные (например, 99% «не фрод», 1% «фрод»).

### Через `class_weight` (для LR)
```python
# Spark не имеет автоматического class_weight, считаем сами
balancing_ratio = train.filter("label = 0").count() / train.count()
train = train.withColumn("class_weight",
    F.when(F.col("label") == 1, balancing_ratio)
     .otherwise(1 - balancing_ratio))

lr = LogisticRegression(weightCol="class_weight", ...)
```

### Undersampling / oversampling
```python
# Простейший undersampling класса 0
n_minor = train.filter("label = 1").count()
n_major = train.filter("label = 0").count()
ratio = n_minor / n_major
majors = train.filter("label = 0").sample(False, ratio)
balanced = majors.union(train.filter("label = 1"))
```

### Метрики
ROC-AUC не зависит от баланса. PR-AUC и F1 — лучше для редких классов.

## Часть 6. Train/Test split

```python
train, test = df.randomSplit([0.8, 0.2], seed=42)

# Стратифицировано (минимум вручную)
fractions = {0: 0.8, 1: 0.8}
train = df.sampleBy("label", fractions, seed=42)
test = df.subtract(train)
```

⚠️ `randomSplit` НЕ стратифицирован. При сильном дисбалансе используйте `sampleBy`.

## Часть 7. Регуляризация

| Тип | Когда |
|-----|-------|
| **L2 (Ridge)** | Стандарт; уменьшает все коэффициенты пропорционально |
| **L1 (Lasso)** | Хорош для разреженности; зануляет неважные признаки |
| **Elastic Net** | Гибрид: `elasticNetParam ∈ [0,1]` |

```python
lr = LogisticRegression(
    regParam=0.1,
    elasticNetParam=0.5,    # 0.5 = пополам L1 и L2
)
```

Большой `regParam` → недо- (underfit). Маленький → пере- (overfit). Подбирать через CV.

## Часть 8. Что писать в конце pipeline'а

```python
def fit_eval(estimator, train, test):
    """Универсальная обёртка для бенчмарка моделей."""
    model = estimator.fit(train)
    pred = model.transform(test)
    auc = BinaryClassificationEvaluator(labelCol="label").evaluate(pred)
    return model, auc

candidates = {
    "LR": LogisticRegression(featuresCol="features", labelCol="label"),
    "RF": RandomForestClassifier(featuresCol="features", labelCol="label", numTrees=100),
    "GBT": GBTClassifier(featuresCol="features", labelCol="label", maxIter=50),
}

for name, est in candidates.items():
    m, auc = fit_eval(est, train, test)
    print(f"{name}: AUC={auc:.4f}")
```

Это стандартный приём — обучить несколько моделей, выбрать лучшую.

## Часть 9. Юридический угол (повторим)

Все три модели — «черные ящики» в разной степени:

- **LR** — самый «белый». Коэффициенты интерпретируются.
- **RF** — feature importance, но конкретное предсказание объяснить сложно.
- **GBT** — то же.

Если модель влияет на **значимые решения** (кредитный скоринг, отказ в страховке, найм), по GDPR Art. 22 и AI Act нужна **объяснимость**. Это разберём в уроке 6.9.
