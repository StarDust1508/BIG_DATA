# Контрибуция в курс

> Если вы прошли курс и хотите его улучшить — добро пожаловать.

---

## Что можно сделать

- 🐛 **Исправить опечатку** — самый простой и ценный вклад.
- 📚 **Дополнить урок** — особенно ценно: примеры из своего опыта.
- 🧪 **Добавить практику** — задачу + решение.
- 🌐 **Перевод** — на английский или другие языки.
- 🔧 **Обновить установку** — если устарели версии (Spark 4.0+, Java 21+).

---

## Как

1. Fork репозитория.
2. Создайте ветку `feature/<краткое-имя>`.
3. Сделайте изменения, проверьте, что markdown и Python-скрипты валидны:
   ```bash
   python -m py_compile path/to/script.py
   ```
4. Откройте Pull Request с описанием:
   - Что меняется?
   - Зачем?
   - Если касается урока — что от этого выиграет читатель?

---

## Стиль

### Уроки
- Заголовок: `# Урок X.Y — название`.
- В начале — краткое описание (1–2 строки).
- Каждый раздел — `## Часть N`.
- В конце — секции `## ✅ Самопроверка` и `## ▶️ Дальше`.

### Код
- Python 3.10+, типы аннотированы.
- Никаких Python-циклов по DataFrame'ам.
- Комментарии на русском.
- Имена переменных — короткие, осмысленные.
- Используйте `ruff format` (или `black`) перед PR.

### Правовая часть
- Ссылки на актуальные редакции актов.
- Без правовой консультации — только информационно.
- Указывать дату последнего изменения соответствующего закона.

---

## О чём пожалуйста не делать PR

- 🚫 «Полная переработка модуля» без обсуждения.
- 🚫 Реклама/курсы/продукты.
- 🚫 Использование платных сервисов в учебных примерах.
- 🚫 Раздувание объёма ради объёма.

---

## Контакты

Создайте Issue для обсуждения больших изменений.

Спасибо, что делитесь знаниями! 🚀
