## 🧩 Тематический охват

### Концепции
- 5V Big Data, экосистема, тренды индустрии.
- CAP, ACID vs BASE, MapReduce-парадигма.
- Форматы хранения: CSV / JSON / Avro / Parquet / ORC.
- Lakehouse: Delta / Iceberg / Hudi.
- Lambda / Kappa архитектуры.

### Инфраструктура
- HDFS, YARN, NameNode SPOF, локальность данных.
- Docker compose для Hadoop-кластера.
- S3 vs HDFS, облачные варианты.
- Spark on YARN / K8s / standalone.

### Python для данных
- Pandas: DataFrame, Series, dtypes, индексы.
- NumPy: ndarray, broadcasting, ufuncs.
- EDA: гистограммы, корреляции, скос.
- Group-by, join, окна, transform.
- Производительность: chunked I/O, Polars, Dask, DuckDB.
- Псевдонимизация, k-анонимность, диф.приватность.

### Apache Spark
- Driver / Executor / Partition / Stage / Task / Shuffle / DAG.
- DataFrame API + Spark SQL + Catalyst.
- Lazy evaluation, transformation vs action.
- AQE (Adaptive Query Execution).
- Тюнинг, skew, salt-trick.
- Caching / persist стратегии.
- Delta Lake: DELETE / UPDATE / MERGE / Time Travel.

### Spark MLlib
- Transformer / Estimator / Pipeline / Model.
- Feature engineering: StringIndexer / OHE / VectorAssembler / Scaler.
- Классификация: LR / RF / GBT.
- Регрессия и кластеризация (KMeans, GaussianMixture).
- Текст: TF-IDF + классификация.
- Cross-validation, ParamGridBuilder.
- Сохранение / загрузка моделей.
- Объяснимость, disparate impact, Model Card.

### Правовая часть
- 152-ФЗ vs GDPR — сравнение.
- AI Act: категории риска.
- Псевдонимизация vs анонимизация.
- Право на забвение в Big Data системах.
- Доказательственная сила ML-выводов в суде.
- Чек-лист compliance для проектов.

## 🌟 Что делает курс «профессиональным»

### 1. Не просто учебник — производственный код
- Все pipeline'ы используют идемпотентность, dynamic partition overwrite, schema validation, DQ-assertions, audit log.
- Структура `extract → clean → enrich → aggregate → load`, как в реальных командах.
- Параметризация по дате через `argparse`.

### 2. Документация на уровне индустрии
- Шаблоны Model Card / Data Card в стандарте Google.
- Юридические разделы в каждом проекте.
- Версионирование моделей и данных.

### 3. Сквозные смысловые связи
- Каждый модуль ссылается на предыдущие.
- Глоссарий и концептуальная карта закрывают «непонятки» из любого места.
- Правовой контекст — сквозной, а не приклеенный сбоку.

### 4. Бесплатная экосистема — полностью
- Все инструменты: Python, Java, PySpark, MLlib, Docker, VS Code — open source.
- Облачные альтернативы: Databricks Community, Colab, Kaggle.
- Готовые конфиги: docker-compose для Hadoop, devcontainer, Dockerfile.
- Никаких платных подписок.

### 5. Quality assurance
- `audit_links.py` — автоматическая проверка ссылок (290 ссылок, 0 битых).
- Все Python-скрипты прошли `py_compile`.
- Smoke-test: ETL модуля 05, MR модуля 03, классификация модуля 06.
