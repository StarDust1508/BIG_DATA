# Урок 3.5 — Hadoop сегодня: что осталось, что ушло

> Технология имеет жизненный цикл. Hadoop был революцией 2006–2015 гг. Сейчас он на спаде. Полезно понимать, что именно осталось.

---

## Часть 1. Что осталось от Hadoop

### ✅ HDFS (как идея и как продукт)

Идея «дёшёвое железо + репликация + локальность» жива. Сам HDFS:
- В on-prem банковских/телеком/гос-кластерах: **активно**.
- В новых проектах: вытесняется S3/MinIO.

### ✅ YARN

Активно в существующих on-prem Spark-кластерах. В облаке вытеснен Kubernetes.

### ✅ Hive

SQL-движок поверх HDFS. Большая инсталлированная база. С Hive 3 + ACID — даже почти Lakehouse. В новом — берут Iceberg+Trino или Snowflake.

### ✅ HBase

NoSQL поверх HDFS для random-access. Нишевая, но живёт.

### ✅ Знание MR-парадигмы

Парадигма «map → shuffle → reduce» — фундамент.

---

## Часть 2. Что фактически умерло

### ❌ Голый MapReduce как фреймворк для нового кода

Никто новые задачи на чистом MR не пишет уже лет 10. Все — на Spark.

### ❌ Pig

DSL поверх MR. Заменён Spark SQL.

### ❌ Импала, Drill, ...

Многие были «убийцы Hive» — большинство забыты.

### ❌ Cloudera, Hortonworks как самостоятельные

Слились в Cloudera в 2019. Бизнес сжимается.

## Часть 4. Когда новый проект всё-таки берёт Hadoop

| Сценарий | Почему |
|---|---|
| Регуляторика «всё в РФ, никакого облака» | Облако недоступно |
| Большой on-prem датацентр, миграция стоит дороже эксплуатации | Sunk cost |
| Узкоспециальное: HBase для random-access, Hive для legacy | Не успели мигрировать |
| Огромные кластера (PB+) с собственным DevOps | Облако дороже |

В **80% новых проектов** Hadoop не появляется. Сразу: S3 + Spark / Snowflake / BigQuery / Databricks.

## Часть 5. Современный «без-Hadoop» стек

```
Источники → Kafka/Kinesis → S3 (Parquet/Iceberg)
                                    │
                                    ▼
                            Spark on K8s / EMR / Dataproc / Databricks
                                    │
                                    ▼
                        Trino / Snowflake / BigQuery → BI / ML
                                    │
                                    ▼
                               MLlib / scikit
```

Никакого HDFS, никакого YARN, никакого Hive. Хотя многие технологии «помнят» Hadoop API.

---

## Часть 6. Что взять из Hadoop-эры в современность

Несмотря на закат, идеи Hadoop остались с нами **навсегда**:

1. **Распределённое хранение** на дешёвом железе с репликацией. (S3 — это та же идея, только managed.)
2. **Локальность вычислений к данным**. Spark тащит code-to-data, как мог Hadoop.
3. **Schema-on-read**. Парадигма Data Lake — наследие.
4. **MR-парадигма** как ментальная модель. Map/Reduce/Shuffle — мы думаем в этих категориях, даже когда не пишем MR-код.
5. **Партиционирование** в Hive-стиле (`/year=2025/month=05/`).

## Часть 7. Карьерный совет

Если вы сейчас в начале пути в Big Data:

✅ **Изучайте:** Spark (обязательно), SQL, Python/pandas, Parquet, Kafka, концепции HDFS поверхностно.
✅ **Полезно:** один любой Lakehouse (Delta / Iceberg), Airflow.
✅ **Бонус:** scikit-learn, MLlib для ML-задач.

⚠️ **Не тратьте много времени на:** голый MapReduce, Pig, Mahout, Oozie, Hortonworks-специфику.
