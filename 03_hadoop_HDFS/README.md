## 🛠 Что нужно установить (бесплатно)

В этом модуле — три варианта на выбор. **Все бесплатные.**

### Вариант A — Docker Compose ⭐ (рекомендую)

Самый быстрый способ получить реальный кластер. Нужен Docker.

1. Поставьте Docker (если нет): [УСТАНОВКА.md §8](../00_введение/УСТАНОВКА.md#8-docker-для-модуля-03) или [среда_разработки.md §3](../ресурсы/среда_разработки.md#3-docker). Альтернативы Docker Desktop: **Rancher Desktop** или **OrbStack** — бесплатны для всех.
2. Конфиги `docker-compose.yml` и `hadoop.env` есть в [среда_разработки.md §3.2](../ресурсы/среда_разработки.md#32-полноценный-hadoop-в-docker-за-5-минут). Положите их в отдельную папку и запустите:
   ```bash
   docker compose up -d
   ```
3. Web UI: http://localhost:9870 (HDFS), http://localhost:8088 (YARN).

### Вариант B — Без Hadoop вообще ⭐

**Главный учебный трюк курса:** MR-парадигма отрабатывается через обычный shell-pipe:
```bash
cat data.txt | python3 mapper.py | sort | python3 reducer.py
```

Это эмулирует `map → shuffle → reduce` без всякого Hadoop. Все наши практики **сделаны так**, чтобы работать в этом режиме. Кластер нужен только если хотите «прочувствовать» инфраструктуру.

Ничего ставить не надо — все скрипты на чистом Python.

### Вариант C — Облако

- **AWS EMR**: free tier на короткое время.
- **Yandex DataProc**: free tier для новых пользователей.
- **Google Dataproc**: $300 credits для нового аккаунта.

Для курса не обязательно. Если хочется «настоящего железа» — это путь.
