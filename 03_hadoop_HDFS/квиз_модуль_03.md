## ✅ Ответы

<details>
<summary>Развернуть</summary>

1. **a.** HDFS + YARN + MapReduce.
2. **b.** NameNode = мета, DataNode = данные.
3. **c.** 128 МБ (иногда 256). Большие блоки = меньше метаданных + последовательное чтение.
4. **c.** 3 копии — баланс между надёжностью и стоимостью.
5. **d.** NameNode — это HDFS, не YARN.
6. **a.** NameNode хранит мета каждого файла → миллион мелких файлов = миллион записей в RAM.
7. **b.** Compute едет к данным, не наоборот.
8. **b.** Spark on YARN — один из стандартных режимов.
9. **b.** Shuffle = перетасовка по ключу + сортировка.
10. **b.** Combiner = мини-reducer. Sum/max/min — да; avg — нельзя без передачи (sum, count).
11. **b.** MR — каждый раунд через диск. Итеративные алгоритмы крайне неэффективны.
12. **b.** Hadoop Streaming абстрагирует язык через stdin/stdout.
13. **a.** Pipe-эмуляция — главный учебный трюк.
14. **b.** Sort нужен, чтобы все записи одного ключа шли подряд.
15. **b.** Неверный результат. Hadoop не проверяет, ассоциативна ли операция.
16. **c.** Object storage (S3/GCS/Azure Blob) — стандарт нового мира.
17. **b.** Lakehouse — ACID-таблицы поверх HDFS/S3.
18. **b.** Перезапись партиции/файла. Lakehouse-форматы делают это «бесплатно».
19. **b.** Encryption zones + Key Management Server.
20. **b.** Голый MR и Pig сегодня — наследие. Время лучше потратить на Spark.

</details>
