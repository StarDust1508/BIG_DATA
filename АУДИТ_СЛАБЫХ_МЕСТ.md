# 🔍 Аудит слабых мест курса

> Честная самокритика. Где наш курс реально проигрывает Coursera Big Data Specialization, OTUS Big Data Architect, ШАД, Databricks Academy и подобным.

> Аудит не для того чтобы расстраиваться — а чтобы видеть, что **закрыть** дополнительными материалами (см. ниже) и куда **расти после курса**.

---

## 🟢 В чём мы СИЛЬНЕЕ многих курсов

Прежде чем критиковать, честно: курс делает несколько вещей **лучше большинства**:

1. **Правовой контекст в каждом модуле** — 152-ФЗ, GDPR, AI Act. У большинства курсов это либо отсутствует, либо вынесено в отдельный «приложение».
2. **Объяснимость и Model Card** — стандарт документации модели у нас встроен. У многих курсов «обучили модель → конец».
3. **Сквозные смысловые связи** — концептуальная карта + глоссарий + ссылки между модулями. Многие курсы — это разрозненные уроки.
4. **На русском с пониманием РФ-специфики** (152-ФЗ, локализация, банковский сектор).
5. **Полностью бесплатный стэк** — все альтернативы есть, ничего платного не нужно.
6. **Самокритичный взгляд** на сам Spark (когда НЕ брать) — большинство курсов продают Spark как «решение всего».

## 🔴 Где мы СЛАБЕЕ — честный список

### 1. ❌ Нет видеолекций
**Реальность:** многие лучше воспринимают видео.
**Что закрываем:** хорошие иллюстрации в markdown + указатели на бесплатные видеокурсы (Databricks Academy, YouTube каналы Databricks/Confluent).
**За рамками курса:** видео мы делать не будем — это другой жанр.

### 2. ❌ Синтетические данные вместо реальных
**Реальность:** Coursera/OTUS дают NYC Taxi, Stack Overflow, MovieLens, реальные банковские датасеты.
**Что закрываем:** в [ДАЛЬШЕ_ПОСЛЕ_КУРСА.md](./ДАЛЬШЕ_ПОСЛЕ_КУРСА.md) ссылки на открытые датасеты + добавим инструкцию «как взять NYC Taxi 170 ГБ и прогнать наш проект 1 на нём».
**В новом материале:** [ресурсы/реальные_кейсы.md](./ресурсы/реальные_кейсы.md) — кейсы из индустрии Netflix/Uber/Spotify/Тинькофф.

### 3. ❌ SQL очень поверхностно
**Реальность:** SQL — главный язык дата-инженера. Мы упомянули, но не углубились.
**Что добавлено:** [ресурсы/глубокий_sql.md](./ресурсы/глубокий_sql.md) — продвинутый SQL для Big Data: window functions advanced, рекурсивные CTE, query optimization, типовые вопросы интервью.

### 4. ❌ Spark Internals слабо
**Реальность:** профи знают про Tungsten, code generation, off-heap, GC tuning. Мы остановились на DataFrame API.
**Что добавлено:** [ресурсы/spark_internals.md](./ресурсы/spark_internals.md) — глубокая теория «что внутри Spark».

### 5. ❌ Streaming только в обзоре
**Реальность:** Spark Structured Streaming + Kafka — отдельная большая тема. Watermarks, exactly-once, late events.
**Что добавлено:** [ресурсы/streaming_kafka_spark.md](./ресурсы/streaming_kafka_spark.md) — мини-курс по streaming с рабочими примерами.

### 6. ❌ Тестирование Spark-кода не показано
**Реальность:** Production-код пишется с тестами. У нас они упомянуты, но не показаны.
**Что добавлено:** [ресурсы/тестирование_spark.md](./ресурсы/тестирование_spark.md) — pytest, chispa, Great Expectations, фикстуры, mock.

### 7. ❌ Вероятностные структуры не упомянуты
**Реальность:** HyperLogLog, Bloom filter, Count-Min Sketch — стандарт инженерного Spark.
**Что добавлено:** [ресурсы/probabilistic_ds.md](./ресурсы/probabilistic_ds.md) — основы + примеры в PySpark.

### 8. ❌ Performance/Profiling — мало живых примеров
**Реальность:** «Spark UI прочитал — нашёл проблему» — это навык. У нас он не отработан.
**Что добавлено:** [ресурсы/performance_practice.md](./ресурсы/performance_practice.md) — реальные сценарии тюнинга с разбором Spark UI.

### 9. ❌ MLOps в зачатке
**Реальность:** MLflow, Feature Store, model registry, A/B тесты, мониторинг drift — это отдельный мир.
**Что добавлено:** [ресурсы/mlops_углублённо.md](./ресурсы/mlops_углублённо.md) — практический MLOps.

### 10. ❌ Подготовки к интервью нет
**Реальность:** курсы вроде «System Design Interview» специально готовят к найму. У нас фокус на навыки.
**Что добавлено:** [ресурсы/интервью.md](./ресурсы/интервью.md) — типовые вопросы DE/ML, system design, SQL-задачи, поведенческие.

### 11. ❌ Jupyter notebook'и не использованы
**Реальность:** многие любят .ipynb за интерактивность + графики inline.
**Что закрываем:** все наши скрипты — обычные .py. В курсе сознательно: production-код должен быть .py. Но в [среда_разработки.md](./ресурсы/среда_разработки.md) есть инструкция, как открыть .py в Jupyter / VS Code Notebook.
**За рамками курса:** не дублируем материал в .ipynb — это раздувание без пользы.

### 12. ❌ Графовые алгоритмы и временные ряды не покрыты
**Реальность:** GraphX, GraphFrames, time series в Spark — отдельные специализации.
**Что закрываем:** только упоминания. В roadmap'е «дальше после курса» — отдельные ссылки.
**За рамками курса:** это специализации, не базовый Big Data.

### 13. ❌ Soft skills для DE/ML не покрыты
**Реальность:** Как читать ТЗ, как презентовать архитектуру, как делать code review — это часть работы.
**Что закрываем:** в [интервью.md](./ресурсы/интервью.md) есть раздел «поведенческие вопросы». Полноценный гайд по soft skills — за рамками этого курса.

### 14. ❌ Cloud-нативно не показано (EMR, Glue, BigQuery)
**Реальность:** В реальной работе вы скорее всего работаете в managed-сервисах.
**Что закрываем:** в [среда_разработки.md §4](./ресурсы/среда_разработки.md#4-облака) указаны Databricks Community, Colab, Kaggle. Для глубины — отдельные курсы провайдеров (AWS, GCP, Azure).
**За рамками курса:** мы учим **PySpark**, который везде одинаков. Cloud-специфику добавляйте под свой проект.

### 15. ❌ Безопасность глубоко не покрыта
**Реальность:** Kerberos, SSO, network isolation, encryption end-to-end — целая дисциплина.
**Что закрываем:** базовое (ACL, encryption at rest) есть. Глубокое — за рамками базового курса.

### 16. ❌ Глубокое обучение не покрыто
**Реальность:** transformers, LLM, fine-tuning, RAG.
**Что закрываем:** упоминания. Это **не** Big Data в классическом смысле — это ML. Отдельный курс.

## 🧠 Что курс НЕ заменит

Будем честными. Курс **не заменит**:

1. **Реальный опыт** в производственной команде.
2. **Mentor'а**, который кропотливо разбирает ваш код.
3. **Live-курсы** с обратной связью.
4. **Сертификации** — наш курс не выдаёт ничего, признанного работодателями.
5. **Конкретный стэк компании** — реальный проект на AWS EMR или Snowflake потребует своих знаний.

Но как **бесплатный фундамент** на русском с правовой составляющей — он сильнее большинства.
