## Часть А. Теория (10 вопросов)

**1.** Что из перечисленного НЕ относится к 5V?
- a) Volume
- b) Velocity
- c) Visibility
- d) Veracity

**2.** Главное преимущество Spark над Hadoop MapReduce — это:
- a) Spark написан на Python
- b) Spark работает с данными в памяти, без промежуточной записи на диск
- c) Spark не требует Java
- d) Spark поддерживает SQL

**3.** В теореме CAP буква «P» означает:
- a) Performance
- b) Persistence
- c) Partition tolerance
- d) Partitioning

**4.** Cassandra относится к категории:
- a) CP (Consistency + Partition tolerance)
- b) AP (Availability + Partition tolerance)
- c) CA (Consistency + Availability)
- d) Полностью ACID

**5.** Колоночные форматы хранения эффективнее строковых для:
- a) OLTP-нагрузок (много мелких записей)
- b) Аналитики (агрегации по колонкам)
- c) Полнотекстового поиска
- d) Хранения изображений

**6.** Lakehouse-формат (Delta, Iceberg, Hudi) добавляет к Parquet:
- a) Колоночное хранение
- b) Сжатие snappy
- c) ACID-транзакции и операции DELETE/UPDATE
- d) Поддержку JSON

**7.** Что произойдёт в Spark при выполнении `df.filter(...).select(...)` без последующего `.show()`?
- a) Запустится вычисление
- b) Файл будет прочитан
- c) Ничего — будет построен план, но не выполнен
- d) Будет ошибка

**8.** Какая операция в Spark обычно НЕ вызывает shuffle?
- a) `groupBy`
- b) `join` (обычный)
- c) `filter`
- d) `distinct`

**9.** «Право на забвение» (Art. 17 GDPR) в Data Lake на голом Parquet:
- a) Реализуется командой DELETE
- b) Невозможно
- c) Возможно, но требует перезаписи всей партиции
- d) Реализуется автоматически

**10.** При обработке ПДн по 152-ФЗ обязательное требование — это:
- a) Только английский язык в коде
- b) Локализация первичной записи на территории РФ
- c) Использование только облачных провайдеров
- d) Открытый исходный код

## Часть В. Открытые (5 вопросов)

**16.** Какие из 5V «выстреливают» в системе биржевой торговли? Объясните почему.

**17.** У вас 80 ГБ данных, помещаются в RAM одной мощной машины. Что лучше — Pandas или PySpark? Почему?

**18.** Чем партиционирование Hive-style (на диске) отличается от партиций Spark в памяти?

**19.** Опишите MapReduce-задачу подсчёта средней зарплаты по департаментам.

**20.** Почему обезличивание (анонимизация) выгоднее делать **до** загрузки в Spark, а не после?

## ✅ Ответы

<details>
<summary>Развернуть ответы (открывайте после своих попыток!)</summary>

**1. c) Visibility.** Правильные V: Volume, Velocity, Variety, Veracity, Value.

**2. b)** Память между шагами — основная причина 10–100× ускорения.

**3. c)** Partition tolerance — устойчивость к сетевому разделению.

**4. b) AP.** Cassandra жертвует мгновенной согласованностью ради доступности и масштабируемости.

**5. b)** Колоночные форматы блестят, когда нужна агрегация по нескольким колонкам без полной прокачки всех строк.

**6. c)** Delta / Iceberg / Hudi добавляют транзакционность поверх Parquet.

**7. c)** Spark — lazy. План есть, выполнения нет до action.

**8. c) filter** не вызывает shuffle (фильтрация локальна). Все остальные — могут.

**9. c)** В голом Parquet файлы иммутабельны → перезапись партиции без удалённой записи. Lakehouse-форматы это делают встроенным DELETE.

**10. b)** Локализация: первичная запись ПДн россиян должна быть на серверах в РФ (242-ФЗ).

**11. Неверно.** Псевдонимизированные данные — всё ещё ПДн. Только полная и необратимая анонимизация выводит их из-под действия 152-ФЗ/GDPR.

**12. Неверно.** Avro — строковый формат, для аналитики «по колонке» проигрывает Parquet/ORC.

**13. Неверно.** 200 МБ переваривает Pandas легко. Big Data начинается, когда данные не помещаются на одной машине.

**14. Верно.** Spark — самостоятельный фреймворк, работает локально, на YARN, K8s, со многими хранилищами.

**15. Неверно.** YARN — часть Hadoop. Spark может работать НА YARN, но это разные продукты.

**16. Velocity (тики приходят миллисекундно), Veracity (нужна проверка на ошибочные сделки), Value (на этом строится бизнес). Volume — относительно среднее. Variety — низкое, всё табличное.**

**17. Pandas, скорее всего.** Spark имеет накладные расходы (JVM, shuffle). На 80 ГБ при достаточной RAM Pandas обычно быстрее. Spark выигрывает, когда: (а) данные не помещаются в память, (б) нужно распараллеливание на много машин, (в) ETL длинный и можно его оптимизировать через Catalyst.

**18. Hive-style партиционирование — это физическое разбиение на папки по значению колонки (`year=2024/month=05/`). Спарк читает только нужные папки. Spark-партиции в памяти — это куски DataFrame для параллельных вычислений. Связаны: при чтении количество in-memory партиций часто = количеству файлов.**

**19. Map:** каждый mapper читает строки сотрудников, выдаёт пары `(department, (salary, 1))`. **Shuffle:** все пары с одинаковым department сгруппировались. **Reduce:** для каждого department сумма зарплат и количество, итог = sum/count.

**20. Несколько причин:** (а) меньше данных в кластере → лучше для производительности; (б) сужаем круг лиц, имеющих доступ к ПДн; (в) при утечке кластера утекают уже обезличенные данные; (г) проще доказать compliance — есть чёткая «граница», после которой ПДн не циркулируют.
