# 🎯 Проект 1 — Детекция аномалий в банковских транзакциях

> Сквозной проект: от сырых CSV до production-ready ML-модели и Model Card.
> Соединяет всё, что мы изучили: ETL, ML, обезличивание, документацию.

---

## 📋 Бизнес-постановка

**Заказчик:** Compliance + Risk-департамент банка.

**Цель:** автоматически отмечать «подозрительные» транзакции для ручной проверки.

**Что есть на входе:**
- Сырые CSV транзакций (200 ГБ в проде; в курсе — синтетика на 1М строк).
- Справочник клиентов с ПДн.

**Что должно быть на выходе:**
- Ежедневный pipeline: помечает транзакции score'ом «аномальности».
- ML-модель `IsolationForest`-подобного типа на распределённых данных.
- Документация: Data Card + Model Card.
- Audit log.

---

## 🏗 Архитектура

```
Источник CSV (transactions.csv + clients.csv)
      │
      ▼
01_load.py        — extract + базовая валидация
      │
      ▼
02_clean.py       — очистка + дедупликация + типы
      │
      ▼
03_features.py    — feature engineering (агрегаты по клиенту, окна)
      │
      ▼
04_pseudonymize.py — псевдонимизация client_id через salted SHA-256
      │
      ▼
05_train.py       — обучение (Random Forest)
      │
      ▼
06_score.py       — инференс на свежих данных
      │
      ▼
Parquet + Model Card + Audit log
```

## ▶️ Запуск

Данные берутся из модуля 02 (`datasets/clients.csv`, `datasets/transactions.csv`):

```bash
python3 02_python_для_данных/практика_1_eda.py
```

Запуск всего pipeline:
```bash
cd 07_проекты/проект_1_аномалии
python3 pipeline.py --run-date 2026-05-15
```

Результат:
- `datasets/anomalies_output/dt=2026-05-15/` — scored транзакции.
- `MODEL_CARD.md` (генерируется автоматически с метриками).
- `audit.log` — кто запустил, что обработал.

---

## 📊 Что будет в Model Card

После запуска `pipeline.py` обновится `MODEL_CARD.md` со свежими:
- Размер train / test.
- ROC-AUC, PR-AUC.
- Confusion matrix.
- Topf-5 признаков по importance.
- Disparate Impact по сегменту клиентов.
- Дата обучения, hash данных, версия Spark.

Образец шаблона: [ресурсы/шаблоны/MODEL_CARD_TEMPLATE.md](../../ресурсы/шаблоны/MODEL_CARD_TEMPLATE.md).

## ⚖️ Правовая часть

Чек-лист, который должен быть выполнен:

- [ ] Согласие клиента на использование транзакций для антифрода (152-ФЗ).
- [ ] Псевдонимизация client_id до того, как данные попадут в ML.
- [ ] Соль из переменной окружения `PSEUDO_SALT`.
- [ ] Audit log: кто, когда, какие модели применил.
- [ ] Human review: подозрительные транзакции **не блокируются автоматически**, а уходят аналитику.
- [ ] AI Act категория: высокий риск (антифрод → значимые последствия для клиента).
- [ ] Подписан Model Card с обоснованием категории риска.
