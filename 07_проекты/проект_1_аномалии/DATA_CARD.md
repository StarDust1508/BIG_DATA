# DATA CARD — Transactions for Anomaly Detection

## 1. Описание
Транзакционные данные клиентов банка для обучения модели детекции аномалий.

## 2. Источник
- Источник: `datasets/transactions.csv` (генерируется в модуле 02, синтетика)
- Метод: синтетический; в реальном проекте — DWH таблица `bank.transactions`
- Частота обновления: ежедневно (учебный режим — на разовом снэпшоте)

## 3. Схема

| Колонка     | Тип       | Описание                         | Nullable |
|-------------|-----------|----------------------------------|----------|
| client_id   | int       | ID клиента — псевдонимизируется в пайплайне | false |
| ts          | timestamp | Момент совершения транзакции     | false |
| category    | string    | Тип операции (4 значения)        | false |
| amount      | double    | Сумма в рублях                    | false |

## 4. Размер
- Строк: ~80 000 (учебная синтетика)
- Размер: ~3 МБ CSV
- Период: январь-апрель 2026

## 5. ПДн
- Содержит ПДн: **ДА** (client_id связан с физлицом через таблицу clients).
- Категория: общие ПДн (по 152-ФЗ).
- Прямые идентификаторы: нет в этом файле, но client_id ссылается на clients.csv.
- Псевдонимизация: производится в pipeline (04_pseudonymize.py) — SHA-256 + соль.

## 6. Качество
- Дубликаты: ~0.1% (синтетика добавила).
- Пропуски client_id: ~2%.
- Опечатки в category: ~0.5% (например, ПЕРЕВОД вместо перевод).

## 7. Доступ
- Где хранится: локально в `datasets/`. В прод — S3 с SSE-KMS.
- ACL: только ML-команда и Compliance.
- Audit log: `audit.log` (создаётся pipeline'ом).

## 8. Жизненный цикл
- Срок хранения сырья: 365 дней (по политике банка).
- Триггер удаления: по истечении срока + по запросу субъекта (GDPR Art. 17, 152-ФЗ).
- Резервные копии: 90 дней.

## 9. Использование
- Модель: `models/anomaly_v1`.
- Downstream: `datasets/anomalies_output/dt=*/` — лента scored транзакций для аналитиков.
